# /plan

触发 planner 代理制定功能实施计划。

## 使用方式

1. 描述你想要的功能
2. 输入 `/plan` 或说"帮我制定一个实施计划"
3. 代理会分析代码库并输出详细计划

## 计划内容

- 功能概述
- 受影响组件（前端/后端/数据库）
- 分步实施清单（含验证方式）
- 风险与缓解措施
- 预估工作量

## 示例

用户：我想添加一个库存预警功能，当库存低于安全库存时自动发送通知。

→ `/plan`

代理输出：
```
## 功能概述
库存预警：当物料库存低于安全库存阈值时触发预警。

## 受影响组件
- 前端: Alerts 页面, Inventory 页面
- 后端: alerts-v1.1.ts, inventory-v1.1.ts
- 数据库: materials 表 (已有 safety_stock 字段)

## 实施步骤
1. 后端：添加预警检查 API → verify: 调用 API 返回预警列表
2. 前端：Alerts 页面展示预警 → verify: 页面显示预警信息
3. E2E：添加预警流程测试 → verify: npx playwright test e2e/alerts.spec.ts
```

## 注意事项

- 计划完成后需用户批准再执行
- 如需调整，可直接告诉代理修改方向
- 简单功能无需完整计划，直接开发即可
