# 主会话审核清单

> 用途: 主会话审核另一台设备扫描报告和后续代码修复。

## 扫描报告审核

| 检查项 | 通过标准 |
|:---|:---|
| 范围正确 | 只扫描任务单列出的 5 个页面 |
| 无代码修改 | `git status --short` 没有新的代码改动 |
| 文件位置明确 | 每个问题都有路径和行号 |
| 严重级别合理 | P0/P1/P2/P3 与影响匹配 |
| 建议可执行 | 能判断是否适合批量修复 |

## 后续修复任务分配规则

适合交给另一台设备:

1. 替换原生 `confirm` 为已有 `ConfirmDialog`。
2. 统一空状态、loading、error 状态。
3. 移除明显硬编码展示数字。
4. 补充简单测试断言。
5. 整理文档和报告。

必须由主会话处理:

1. 公共 hook 或组件设计。
2. 跨页面状态同步方案。
3. 权限、认证、用户角色。
4. 后端接口协议。
5. 数据一致性和库存数量变更。
6. E2E 或 CI 结构调整。

## 合并前检查

| 检查项 | 命令或方式 |
|:---|:---|
| 确认只包含任务范围文件 | `git diff --name-only` |
| 检查无明显空白错误 | `git diff --check` |
| 前端类型和构建 | `cd 前端代码 && npm run build` |
| 相关单测 | `cd 前端代码 && npm test -- --runInBand` 或按项目实际命令 |
| 工作日志更新 | `.claude/SESSION-B-WORKLOG.md` |

## 通过标准

只有同时满足以下条件才允许进入提交:

1. 文件范围符合任务单。
2. 没有引入假功能。
3. 没有绕过真实 API。
4. 没有修改禁止文件。
5. 测试或人工验证结果明确。
6. 残留风险已写清楚。
