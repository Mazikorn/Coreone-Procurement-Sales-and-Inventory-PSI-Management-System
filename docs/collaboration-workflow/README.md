# COREONE 双设备协作工作流

> 版本: 2026-06-12
> 目的: 让主设备负责任务规划、复杂实现和质量审核，另一台设备负责简单但量大的执行任务。

## 协作角色

| 角色 | 设备 | 主要职责 |
|:---|:---|:---|
| 主会话 | 当前 Codex 设备 | 工作拆分、复杂实现、公共组件、最终审核、合并前验证 |
| 执行会话 | 另一台设备 | 批量扫描、简单页面修复、文档整理、测试补充 |

## 今日目标

2026-06-12 的第一次协作目标是先建立协作基线，不直接扩大代码改动面。

1. 主会话审核当前批次2产出，包括通用组件、入库拆分、库存和盘点改造。
2. 执行会话只做只读扫描，不修改代码。
3. 执行会话提交扫描报告后，由主会话决定下一批可执行修复任务。

## 执行原则

1. 先读根目录 `AGENTS.md`。
2. 先读 `.claude/SESSION-B-WORKLOG.md`。
3. 没有明确任务单时，不主动改代码。
4. 另一台设备默认只处理简单、重复、边界清楚的工作。
5. 所有修改都必须说明修改文件、测试结果和残留风险。
6. 主会话负责最终质量门禁。
