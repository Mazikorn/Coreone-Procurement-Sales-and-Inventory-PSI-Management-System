# 出库管理 主流程收口记录

## 入口边界

- 入口: 出库管理
- 导航路径: 库存作业 -> 出库管理 (`/outbound`)
- 主要用户: 仓管、技术员
- 本轮目标: 检查并修复普通出库、BOM 出库、LIS/项目出库形成库存扣减、批次成本、病例/项目成本事实的高频主流程核心问题。
- 包含范围: 出库列表、普通出库、BOM 出库、批次扣减、库存流水、项目/病例/BOM/成本异常联动。
- 明确不做: 不提前修退库、供应商退货、调拨、报废、盘点本体；若出库发现这些入口问题，先作为相邻线索记录。

## 当前检索记录

- 从 `014 库存列表` 自动切换而来。
- 已检索出库前后端实现、页面表格/表单、权限矩阵、历史 FRS/TS 线索、现有出库测试和出库端到端测试。
- 当前实现中普通出库、BOM/LIS 出库均会扣减库存、批次和库位库存，并写入 `stock_logs`；BOM 出库还会写入 `outbound_abc_details` 并进入成本异常链。
- 现场发现通用出库入口同时暴露/接受 `transfer`、`scrap` 类型，但调拨、报废在导航队列中有独立入口，且需要目标库位/报废原因等专用业务字段。
- 现场发现 BOM 出库记录在列表中暴露“编辑”，前端提交时可能走新增 BOM 出库或普通 PUT，后端也允许把 BOM 出库单改写为普通出库单。

## 产品建模质疑

| 检查点 | 当前设计 | 质疑结论 | 影响 | 处理决定 |
|:---|:---|:---|:---|:---|
| 分类是否准确 | `/outbound` 同时作为项目出库、调拨出库、报废出库入口 | 调拨/报废不是普通出库子类型，应走专用入口 | 会绕过目标库位、报废原因、撤销口径等专用事实 | P1 修复：通用出库只允许项目出库；调拨/报废回到 018/019 |
| 字段是否够用 | 普通出库字段能表达项目领用；BOM 出库字段能表达项目/BOM/病例/样本数 | 普通出库不应承载调拨/报废，因为字段不足 | 调拨/报废若从此入口写入，会形成业务事实错误 | 已修复入口和 API |
| 关系是否正确 | BOM 出库同时关联 BOM 快照、病例聚合和 ABC 明细 | BOM 出库单不能普通编辑，否则成本快照和库存明细会错位 | 后续成本分析可能双扣、漏扣或保留错误快照 | P1 修复：BOM 出库普通编辑 409，页面隐藏编辑 |
| 状态流转是否合理 | 当前直接出库为 completed；取消/删除走库存回滚 | 主流程可接受；低频 pending 取消状态本轮不扩展 | 不阻断项目/BOM 出库主流程 | 保留，后续若恢复待审批流再独立评估 |
| 用户工作量是否合理 | 项目/BOM/LIS 出库在一个入口；调拨/报废另有入口 | 收敛后更符合导航分工，减少误操作 | 用户不再需要长期人工纠错错误类型单据 | 已修复 |
| 是否支撑成本分析 | BOM 出库按批次价写 ABC 明细，普通项目出库保留批次成本 | 支撑成本分析；未配置收费映射时进入成本异常链 | 成本异常可被后续成本治理入口处理 | 已验证 |
