# 主流程导航入口清单

> 创建日期: 2026-06-20
> 来源: `前端代码/src/components/layout/AppSidebar.tsx`
> 用途: 目标模式自动选择、收口和切换导航入口

## 一、使用规则

1. 本清单按导航栏功能入口维护，不按按钮、弹窗、API 或测试文件拆分。
2. `status` 可为 `pending`、`in_progress`、`completed`、`blocked`。
3. 执行者每次只能有一个 `in_progress` 入口。
4. 入口完成后，必须填写 `run_doc`、`completed_at` 和 `verification_summary`。
5. 如果实际导航栏发生变化，执行者必须先复核 `AppSidebar.tsx` 并更新本清单。
6. 本清单不是问题池，不预设缺陷；具体问题只写入 `docs/main-flow-entry-runs/` 下的入口工作记录。

## 二、优先顺序原则

入口按业务形成顺序推进:

1. 基础数据配置: 先确认分类、主数据、BOM、收费和成本配置是否能表达真实业务。
2. 过程数据: 再确认采购、入库、库存、出库、退库、退货、调拨、报废、盘点、对账等过程事实是否可信。
3. 成本分析: 最后确认成本看板、切片成本、盈利分析、收费对照、趋势和仪表盘是否基于可信事实得出可解释结果。
4. 审计治理: 最后收口用户、权限、日志等治理入口，确保系统使用和追溯可控。

每个入口都必须检查产品建模，不只检查功能异常。尤其是基础数据阶段，必须质疑分类、字段、关系和状态是否准确。

## 四、隐藏或子入口处理规则

以下路由当前存在于 `App.tsx`，但没有作为左侧导航栏的一级菜单展示:

- `/equipment/types`
- `/equipment/depreciation`
- `/abc/cost-drivers`
- `/abc/budgets`
- `/abc/quality-costs`
- `/abc/variance`
- `/abc/alerts`
- `/abc/audit`
- `/abc/quarterly-adjustment`
- `/abc/personnel-efficiency`
- `/abc/model-validation`
- `/indirect-costs`

处理规则:

1. 如果它们能从某个导航入口内部进入，就作为该导航入口的包含范围处理。
2. 如果页面不可从导航入口进入，先记录为相邻入口线索，不因隐藏路由打断当前入口。
3. 如果后续产品决定把它们加入导航栏，再补入本入口队列。
4. 隐藏或子入口也必须接受产品建模质疑，不能因为它不是一级菜单就只做功能 bug 检查。
