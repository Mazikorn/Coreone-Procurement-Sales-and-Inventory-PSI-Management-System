## 1. MOD-01 认证与登录

### AC-01-001 用户名密码登录

**前置条件**：系统已启动，存在有效用户 admin/admin123

**操作步骤**
1. 打开浏览器访问 `http://localhost:8080`
2. 输入用户名 `admin`，密码 `admin123`
3. 点击"登录"

**期望结果**
- 页面跳转到仪表盘 `/`
- 侧边栏显示完整菜单（admin 全部权限）
- 顶部显示用户名和角色

**数据验证**
- localStorage 中存在 `token` 和 `user`
- `user.role === 'admin'`

### AC-01-002 Token 刷新

**前置条件**：已登录状态

**操作步骤**
1. 等待 Access Token 过期（8 小时）或手动清除 Access Token
2. 执行任意 API 操作

**期望结果**
- 自动通过 Refresh Token 获取新 Access Token
- 操作正常完成，无中断

### AC-01-003 用户登出

**前置条件**：已登录状态

**操作步骤**
1. 点击顶部栏"登出"按钮

**期望结果**
- 跳转到登录页
- localStorage 中 token 和 user 已清除
- 访问任何业务页面被重定向到登录页

## 4. MOD-08 入库管理

### AC-08-001 创建直接入库

**前置条件**：以 warehouse_manager 登录，已有物料

**操作步骤**
1. 打开"入库管理"页面
2. 点击"新建入库"
3. 选择入库类型"直接入库"
4. 选择物料
5. 输入批次号"BATCH-001"
6. 输入数量 100
7. 输入单价 50
8. 选择库位
9. 点击"保存"

**期望结果**
- 入库单创建成功，单号 IB-yyyymmdd-xxx
- 库存增加 100
- 批次记录创建（remaining=100）
- 采购订单不受影响

### AC-08-002 采购入库关联采购订单

**前置条件**：已有采购订单 PO-001（ordered_qty=200）

**操作步骤**
1. 创建入库单，选择"采购入库"
2. 关联采购订单 PO-001
3. 入库数量 100
4. 保存

**期望结果**
- 入库单创建成功
- 采购订单 received_qty=100，状态变为 partial
- 再入库 100，采购订单状态变为 completed

### AC-08-003 取消入库

**前置条件**：已有入库单（状态 completed）

**操作步骤**
1. 选择入库单
2. 点击"取消"
3. 输入取消原因

**期望结果**
- 入库单状态变为 cancelled
- 库存扣减
- 批次 remaining 扣减
- 采购订单 received_qty 回退

### AC-08-004 删除入库前检查

**前置条件**：入库单对应的批次已有出库记录

**操作步骤**
1. 尝试删除该入库单

**期望结果**
- 提示"该批次有出库记录，无法删除"
- 入库单未被删除
