# COREONE G1 收入口径 · 纯实验室收入分类规则

> **版本**: 1.0 | **日期**: 2026-06-30 | **口径**: 纯实验室技术收入（不含医生诊断/报告/对方现场）
> **产出方式**: 和睦家真实收费单（单据+对账单）行级代码逐条核 → 用户逐项确认边界。
> **用途**: 院级盈亏（Phase 3）的收入侧口径 + 收入监管基线。配套 G2 成本口径。

---

## 0. 两条边界原则（用户确认）

1. **IN = 实验室做的技术工序**（制片：取材/脱水/包埋/切片/HE；染色：免疫组化/特殊染色）。**医生诊断、报告 = OUT**（国标"病理诊断费" vs "病理标本处理费"）。
2. **且仅限康湾自己实验室做的**——在**对方现场**做的（不耗康湾成本，如冰冻现场服务）= **OUT**。

> 一句话：**纯实验室收入 = 康湾实验室亲自完成的制片+染色技术部分。**

---

## 1. 完整分类规则（= 分类引擎要实现的）

| 项目 / 代码 | 归类 | 处理方式 |
|---|---|---|
| **检查与诊断**（2703：手术标本 270300005 / 内镜活检 270300002） | **拆** | 老捆绑码 = 制片(IN) + 诊断(OUT)，按方案2拆（见 §2）。国标处理费率=组织 ¥36/标本 |
| **冰冻切片检查与诊断 / 术中快速冰冻诊断**（¥180/标本，扣率0.8） | **拆** | 同检诊（组织 ¥36/标本） |
| **妇科TCT检测**（¥150/玻片，扣率0.7） | **拆** | 细胞制片(IN) + 阅片(OUT)，细胞处理费率 ¥75/玻片 |
| **免疫组化 / 特殊染色**（2705：270500002 / 270500001） | **整条 IN** | 染色技术；判读已含在诊断费里，不重复计 |
| **组织学报告**（2709：270900099，扣率1.0） | **整条 OUT** | 医生/报告（和睦家为外籍双语报告附加费） |
| **医生及技术员冰冻诊断现场服务费**（¥1500/次，扣率1.0） | **整条 OUT** | 对方现场制片、不耗康湾成本（原则2） |
| **新国标拆分码**（如 012100000030000 标本处理费 / 012100000010000 诊断费） | **直接用** | 处理费=IN、诊断费=OUT，已分开，无需拆 |

> **辅助信号（非硬规则）**：扣率=1.0 不打折的项（报告、现场服务费）多为医生/服务费；扣率 0.8 多为技术/处理。但"术中冰冻诊断"是 0.8 却含诊断，故仍按上表逐项判。

---

## 2. 拆分方法（方案2 · 按标本数，用户选定）

老捆绑码按「国标标本处理费(每标本) : 诊断费(每病例)」比例拆，**逐病例**：

```
制片份额(IN) = Σ(处理费率ᵢ × 标本数ᵢ) / ( Σ(处理费率ᵢ × 标本数ᵢ) + 诊断费105 )
诊断份额(OUT) = 1 − 制片份额
IN金额 = 病例内捆绑行结算总额 × 制片份额
```

- 处理费率：组织 ¥36/标本、细胞 ¥75/玻片、（冰冻按组织 ¥36 近似）。诊断费 ¥105/病例（一病例判读一次）。
- **为什么按标本数**：处理随标本/蜡块数增加，诊断按病例固定 → 多标本病例制片占比自然更高（比固定比例更准）。
- 标本数 ≈ 该病例该类捆绑行的行数（1 部位/标本/玻片 ≈ 1 行）。

## 3. 手核答案锚（真实数据，守恒校验通过）

| 锚 | 纯实验室 IN | 守恒 |
|---|---:|---|
| **和睦家全月 26.2**（结算表 × LIS 真蜡块精算，165 病例 **100% 匹配**） | **¥27,870**（占 50.2%） | 55,541 ✓ |
| 和睦家 W4（单据2026062607544902，25 病例；数量估算·无该期 LIS，次锚/下限） | ¥7,118 | 13,152 ✓ |

**精度进展（全月，越用真系统数据越准）**：

| 口径 | 纯实验室 | 说明 |
|---|---:|---|
| 数量-忽略（每行=1标本） | 22,835 | 最粗 |
| 数量-aware（账单数量） | 25,772 | 中 |
| **LIS 真蜡块** | **27,870** | 最准；估算比它低 ~22% |

**LIS 双核验（同期同院对账单 × LIS，通过）**：免疫组化 LIS **80** ≈ 对账单反算 **79**；特染 LIS **2** = 对账单 **2**；蜡块 LIS **260** ＞ 账单部位 ~150（一个部位切多个蜡块，故制片应按 LIS 蜡块拆，不是账单部位/数量）。

> 制片份额（最终）= `36 × LIS蜡块数 / (36 × LIS蜡块数 + 105)`，**逐病例**（对账单 × LIS 按病理号 join）。W4 ¥7,118 是无 LIS 的数量估算，为下限次锚。

## 4. 产品级影响（重要）

**原"实验室收入"口径（全月 ¥55,541）把医生诊断/报告/现场服务都算进去了，真·纯实验室仅 ¥27,870（占 50.2%）——高估约 2 倍。** 院级盈亏收入侧、现有看板的"实验室收入"都要按本口径重算。

---

## 5. Phase 2 实现要求（本口径给分类引擎的输入）

现有分类器只会「整行 IN/OUT」，实现本口径需新增：
1. **「一码按比例拆 IN/OUT」能力**（方案2 公式）——现在没有。
2. **国标处理费率表**（组织 ¥36/标本、细胞 ¥75/玻片、分子 ¥100/次…）+ 诊断费 ¥105/病例。
3. **老码 / 新码识别**：老捆绑码→拆；新国标拆分码→直接用。
4. **「对方现场」标记**：现场服务类 → OUT（不耗康湾成本）。
5. 逐院配置：识别词/费率可逐院覆盖（不同医院码/价不同）。
6. **收入结构 = 4 归类**：`in`(实验室) / `split`(拆) / **诊断桶**(我们的钱非实验室：报告+诊断+现场服务) / **外送桶**(非我们的钱：NGS/FISH/共建分成)。守恒：四类 + 未匹配 + 歧义 = 声明合计。
7. **⭐ 架构：制片按 LIS 蜡块拆 → pipeline 必须「对账单 × LIS 按病理号 join」**。制片份额用 LIS 真蜡块（不是账单数量）。LIS 还提供 G2 成本要的真工作量（蜡块/免疫组化/特染 数）。这比"纯分类器拆分"大——收入计算依赖 LIS 数据。

---

## 6. 与 G2 成本口径的一致性（闭环）

G2 成本也**只算制片/染色工序、不含诊断** → 收入只算制片、成本只算制片，**口径一致**。医生诊断在本系统**既无收入也无成本**。G3 毛利 = G1(制片+染色收入) − G2(制片+染色成本)，干净闭环。
