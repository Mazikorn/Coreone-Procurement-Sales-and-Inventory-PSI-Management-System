## 一、现状诊断

### 1.1 出库成本（已完善）

`outbound_items` 已完整保存批次级成本：
- `batch_id` / `batch_no`：实际消耗批次
- `unit_cost` / `total_cost`：批次实际单价和金额
- `outbound_records.total_cost`：出库单合计

**缺口**：同一物料跨多批次时，前端仅展示明细行，未展示聚合计算过程。

### 1.2 退库（无成本追踪）

`return_records` 表结构：
```sql
id, return_no, material_id, batch_id(NULL), quantity, reason, operator, status, remark
```

- 创建时仅扣减库存总量，不关联具体批次
- 无 `unit_cost` / `total_cost` 字段
- 删除时仅恢复总量

**影响**：退库成本无法追溯到具体批次，成本报表中退库数量被忽略，导致"出库成本虚高"。

### 1.3 供应商退货（无成本追踪）

`supplier_returns` 表结构：
```sql
id, return_no, material_id, batch_id(NULL), batch_no(NULL), quantity, ...
```

- `batch_id` / `batch_no` 始终插入 `NULL`
- `refund_amount` 用户手动填写，无校验
- 无自动成本回填

**影响**：退货成本无法与入库批次关联，无法准确计算"有效采购成本"。

### 1.4 BOM 理论成本 vs 实际成本（有数据，无展示）

阶段3已实现：
- `bom-v1.1.ts` GET /:id 时实时计算理论成本（`Σ(usage × 批次加权平均价)`）
- `outbound-v1.1.ts` POST /bom 保存了实际出库的批次级成本

**缺口**：前后端都有数据，但用户看不到"理论 vs 实际"的对比，更看不到差异原因。

## 二、行业调研验证

### 2.1 FEFO 批次分配的行业依据

**调研结论**：医疗行业（特别是IVD试剂、药品、疫苗、高值耗材）**普遍采用 FEFO（先到期先出）**。

> "药房、检验科、血库、高值耗材库采用 FEFO；普通仓库/低值耗材采用 FIFO。"
> —— [医院物资管理最佳实践](https://www.zelthy.com/blog/hospital-inventory-management-best-practices)

> "IVD 试剂在入库时为每个最小包装生成唯一识别码（含品类、规格、批号、效期），出库时以最小包装为单位扫码扣减，实现批次级精准库存记录。"
> —— [基于唯一识别码的医用体外诊断试剂管理](https://zgylqxzz.xml-journal.net/cn/article/pdf/preview/10.12455/j.issn.1671-7104.230438.pdf)

**反向验证**：阶段2/3采用的 FEFO + 加权成本计算 ✓ **符合行业惯例**

### 2.2 退库成本冲减的行业依据

**调研结论**：SPD 系统和医院成本核算中，退库必须按原成本冲减。

> "退库成功后，系统按原入库批次成本冲减科室当月耗材成本，避免'已退库仍承担成本'的核算误差。"
> —— [SPD物资供应链管理平台使用说明书](https://portal-data-cn.obs.cn-north-4.myhuaweicloud.com/marketplace/public/app/attachment/20191228/ca85f72c-f91e-41e8-8466-ae09430b37e2/1912280612572024.pdf)

> "病理科试剂的特殊退库逻辑：温控脱链即锁定退库、仅允许未拆封整盒退库。"
> —— [SPD 模式下 IVD 耗材管理](https://portal-data-cn.obs.cn-north-4.myhuaweicloud.com/marketplace/public/app/attachment/20191228/ca85f72c-f91e-41e8-8466-ae09430b37e2/1912280612572024.pdf)

> "ERP 系统维护一个按时间顺序排列的成本堆栈，销售退库时按原发出时所结转的成本将存货回填至库存。"
> —— [库存成本核算方法实务](https://www.chanjet.com/lker/652c8f95e4b05985f845a38f.html)

**反向验证**：退库必须追踪成本、关联批次 ✓ **行业标配**

### 2.3 差异分析（价格差异 + 用量差异）的制度依据

**调研结论**：标准成本法及差异分析已被医院成本管理制度明确采纳。

### 2.4 供应商退货绑定批次的行业依据

**调研结论**：高值耗材和IVD试剂退货必须追溯到具体批次。

> "每件耗材具有唯一条码，记录名称、规格型号、批号、效期、使用患者等信息，实现从入库→临床使用→术后跟踪→回收/重用/报废的全流程闭环监管。"
> —— [高值耗材管理核心产品 HVCM](http://www.fortuneltd.com/html/hvcm.html)

> "退货审核通过后生成退货单，实物退回供应商，系统在财务与库存模块同步扣减。"
> —— [SPD 物资供应链管理平台](https://portal-data-cn.obs.cn-north-4.myhuaweicloud.com/marketplace/public/app/attachment/20191228/ca85f72c-f91e-41e8-8466-ae09430b37e2/1912280612572024.pdf)

**反向验证**：供应商退货强制选批次 ✓ **行业追溯要求**

### 2.5 LIFO 在中国会计准则中的限制（关键修正）

**调研发现**：

> "根据中国《企业会计准则第1号——存货》（2006年起），企业应当采用先进先出法、加权平均法或个别计价法确定发出存货的实际成本。后进先出法（LIFO）在中国会计实务中已不再适用。"
> —— [先进先出法与后进先出法区别](https://www.chinaacc.com/zhucekuaijishi/jhwd/zy20240118121917.shtml)

**重要修正**：
- 本项目中的"出库"不是对外销售，而是**内部领料/消耗**
- 退库也不是"销售退回"，而是**领料退回仓库**
- 对于**领料退库**，行业实务中的处理原则是：**按原发出成本追溯退回**
- 算法上从最近出库记录查找，是因为"用户通常退最新领的"——这是**业务匹配逻辑**，不是改变存货计价方法

**修正后的表述**：退库采用"**原发出成本追溯法**"——查找该物料最近出库记录，按原出库时的 unit_cost 确定退库成本。这符合个别计价法的原则（能追溯到具体批次的具体成本），也符合ERP实务中的"按原结转成本冲回库存"。

### 2.6 单张切片成本核算的行业实践

> "病理科作为医技科室，其项目成本核算通常采用作业成本法（ABC法）：将制片流程分解为标本接收、固定、脱水、包埋、切片、染色、封片、阅片诊断等作业环节，按各环节资源消耗归集到单个项目。"
> —— [临床分子病理实验室的建立－标准化管理](http://medi-guide.meditool.cn/guidepdf/57806CAE-A942-A49A-3BCF-B444DA71365B.pdf)

> "IVD 项目成本效益模型：P = m × Pm - Σ(li × ui) - C，其中 li 为第 i 种试剂用量，ui 为试剂单价（按入库批次加权平均或先进先出）。"
> —— [Research on Intelligent Big Data Cost Analysis](http://cs.china-cmd.org/zgylsb/CN/article/downloadArticleFile.do?attachType=PDF&id=16666)

**反向验证**：单项目/单切片成本 = Σ(物料用量 × 批次单价) + 分摊成本 ✓ **与阶段3的公式一致**

## 三、核心设计决策（经调研验证）

### 决策1：退库成本采用「原发出成本追溯法」

**问题**：退库时如何确定成本？

| 方案 | 说明 | 调研验证 |
|------|------|---------|
| 用户指定批次 | 用户选择退哪一批 | 操作复杂，增加认知负担 |
| **原发出成本追溯** | **查找最近出库记录，按原出库 unit_cost 确定退库成本** | **✓ ERP实务标准做法；✓ 符合个别计价法原则** |
| 当前加权平均价 | 用当前库存均价 | 与出库成本不匹配，导致报表不平衡 |

**决策**：原发出成本追溯法。查找该物料最近 `outbound_items` 记录，按原出库的 `unit_cost` 确定退库成本。

**算法说明**：
- 按出库时间倒序查找（因为用户通常退最新领的）
- 找到后：退库的 `unit_cost` = 原出库记录的 `unit_cost`
- 同时记录 `batch_id` = 原出库记录的 `batch_id`
- 这是**个别计价法**的实践形式，符合中国会计准则

**边界**：如果找不到出库记录（如直接入库的物料从未出库过），兜底方案：使用当前库存加权平均价。

### 决策2：供应商退货必须绑定批次

**问题**：供应商退货时是否强制选批次？

| 方案 | 说明 | 调研验证 |
|------|------|---------|
| 不强制 | 只选物料和数量 | 简单，但成本无法追溯 |
| **强制选批次** | **从有库存的批次中选择** | **✓ 高值耗材/IVD追溯要求；✓ SPD标准流程** |

**决策**：强制选批次。创建界面从 `batches` 表中选择 `remaining > 0` 的批次，自动回填 `unit_cost` = `inbound_price`。

### 决策3：差异归因采用「价格差异 + 用量差异」双维度拆解

**问题**：BOM 理论成本 vs 实际成本出现差异，如何归因？

标准管理会计差异分析模型（已被《上海市医院成本管理暂行办法》采纳）：

```
总差异 = 实际成本 - 理论成本
       = 价格差异 + 用量差异

价格差异 = Σ(实际用量 × (实际单价 - 理论单价))
用量差异 = Σ(理论单价 × (实际用量 - 理论用量))
```

**调研验证**：
- ✓ 亚辉龙IPO数据显示 IVD 原材料成本占比 73%-85%，存在显著的价格差异和用量差异
- ✓ 《上海市医院成本管理暂行办法》明确标准成本法可用于医院成本控制
- ✓ 实验室试剂存在"同样试剂在不同厂家采购后价格区别明显"的现象

**适用场景**：

| 差异类型 | 典型原因 | 用户行动 |
|---------|---------|---------|
| 价格差异为正 | 最近采购涨价、换用高价品牌 | 评估是否锁定长协价、评估品牌池配置 |
| 价格差异为负 | 找到更便宜供应商、批次促销 | 采购策略有效 |
| 用量差异为正 | 操作损耗大、重复实验、标准用量不准 | 优化 SOP、培训操作员、下调 BOM 标准 |
| 用量差异为负 | 实际用量低于标准 | 下调 BOM 标准用量 |

**决策**：在项目成本详情中展示这两个维度，帮助用户定位问题。

### 决策4：成本明细展示以「物料聚合」为单元

**问题**：出库详情中同一物料跨多批次，怎么展示最清晰？

| 方案 | 说明 | 优劣 |
|------|------|------|
| 仅明细行 | 每批次一行 | 简单，但用户需心算聚合 |
| **明细行 + 聚合汇总** | **先展示物料汇总，可展开看批次明细** | **清晰，用户一眼看到物料总成本** ✅ |

**决策**：OutboundDetailModal 改为"物料聚合"视图，同一物料跨多批次时显示汇总行，附带"展开"按钮查看批次明细。

### 4.4 退库/退货成本回溯

#### 4.4.1 退库成本追踪（returns-v1.1.ts）

**数据库变更**：
```sql
ALTER TABLE return_records ADD COLUMN unit_cost DECIMAL(18,4) DEFAULT 0;
ALTER TABLE return_records ADD COLUMN total_cost DECIMAL(18,4) DEFAULT 0;
ALTER TABLE return_records ADD COLUMN batch_id TEXT;
ALTER TABLE return_records ADD COLUMN batch_no TEXT;
ALTER TABLE return_records ADD COLUMN outbound_item_id TEXT; -- 关联原出库记录
```

**后端逻辑变更**：
1. POST / 创建退库时：
   - 查找该物料最近的 `outbound_items` 记录（按 `created_at DESC`）
   - 匹配规则：找到 `quantity >= 退库数量` 的记录（或跨多条合并）
   - 回填 `batch_id`, `batch_no`, `unit_cost` = 该出库记录的 unit_cost
   - `total_cost` = `quantity × unit_cost`
   - 如果最近出库量不足，继续往前找（多条出库记录合并）
   - 兜底：无出库记录时使用当前库存加权平均价

2. DELETE /:id 撤销退库时：
   - 恢复对应 `batch_id` 的 `remaining` 库存（不只是 inventory.stock）

**前端变更**：
- 退库列表展示 `total_cost`
- 退库详情展示批次信息和成本

#### 4.4.2 供应商退货成本追踪（supplier-returns-v1.1.ts）

**数据库变更**：`supplier_returns` 已有 `batch_id` 字段，无需新增。

**后端逻辑变更**：
1. POST / 创建退货时：
   - **前端传入 `batchId`**（强制）
   - 后端验证该批次 `remaining >= quantity`
   - 自动填充 `batch_no` = 批次批号
   - 自动填充 `unit_cost` = 批次 `inbound_price`
   - `refund_amount` 默认 = `quantity × unit_cost`，允许用户修改

2. DELETE /:id 删除退货时：
   - 恢复对应 `batch_id` 的 `remaining` 库存

**前端变更**：
- 创建表单增加"批次选择"（SearchableSelect，只显示有库存的批次）
- 选择批次后自动显示"入库单价"和"建议退款金额"
- 列表展示 `unit_cost` 和 `refund_amount`

#### 4.4.3 成本报表整合

**reports-v1.1.ts 调整**：
- `/reports/cost-by-project`：总成本 = 出库总成本 - 退库总成本

## 五、数据流图示

### 退库成本回溯（原发出成本追溯法）

```
创建退库
    │
    ▼
查询 outbound_items
  WHERE material_id = ?
  ORDER BY created_at DESC
    │
    ▼
匹配最近出库记录
  ├── 找到 → batch_id = 出库批次
  │           unit_cost = 出库单价（个别计价）
  │           total_cost = 退库数量 × unit_cost
  │
  └── 未找到 → 使用库存加权平均价（兜底）
    │
    ▼
写入 return_records
    │
    ▼
扣减 inventory.stock
扣减 batches.remaining（对应批次）
```

### 供应商退货成本回溯

```
创建退货
    │
    ▼
前端选择 batch_id
    │
    ▼
后端读取 batches.inbound_price
  → unit_cost = inbound_price
  → refund_amount 默认 = quantity × unit_cost
    │
    ▼
写入 supplier_returns
    │
    ▼
扣减 inventory.stock
扣减 batches.remaining（对应批次）
```
