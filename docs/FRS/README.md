## 阅读指引

1. **新读者**: 先阅读 `docs/00_Project_Governance_Framework.md` 和 `docs/14_Project_Resume_Report.md`，再阅读各模块 FRS。FRS-00 当前缺失，不能作为入口。
2. **按模块查阅**: 每个功能模块独立成文，包含该模块的全部 API 详情、字段规则、业务流程和测试要点。
3. **跨模块关联**: 文档中通过链接引用相关模块，如物料管理依赖物料分类的编码规则。

## 文档规范

每个 FRS 文档均包含以下章节：
- **功能概述**: 模块定位和访问权限
- **业务流程图**: Mermaid 流程图展示核心业务流程
- **API 列表**: 模块全部接口清单
- **接口详情**: 每个接口的请求参数、响应字段、业务流程规则
- **隐含规则显式化**: 将代码中的隐含业务规则明确文档化
- **数据模型**: 实体定义和关联关系
- **交互细节**: 前端交互和异常处理矩阵
- **测试要点**: 该模块的核心测试场景

## 数据来源

字段定义和规则来自后端路由源码、历史 FRS 生成结果和测试场景的逆向整理。由于当前工作树中存在 `app.ts` 挂载但路由文件缺失的情况，本目录不能声明“与实现完全一致”；使用前需按当前代码重新核对。
