# ABC 成本核算 — 产品功能定义 v4.0

> **版本**: v4.0  
> **日期**: 2026-06-04  
> **状态**: 基于用户原则的重大迭代  
> **核心原则**: 最小系统使用成本 + 数据准确 + 运营管理价值

---

## 一、设计原则（v4.0 核心）

### 用户明确的原则

1. **最小系统使用成本**：用户不愿意在系统里记录太多数据
2. **数据准确优先**：自动关联的数据不准，不如不记录
3. **运营管理价值**：提供有用的管理视图，但不要增加用户负担

### 推导出的设计准则

| 准则 | 说明 | 反例 |
|------|------|------|
| **能算则不录** | 能通过公式计算的数据，不要让用户录入 | 设备使用记录 |
| **能用则不细** | 粒度够用即可，不要过度细化 | 标准工时到检测项目 |
| **能简则不繁** | 流程能简化就简化，不要过度审批 | 季度调整多级审批 |
| **能准则不猜** | 数据不准不如不提供，避免误导决策 | 自动关联设备使用 |

## 二、v3.0 → v4.0 变更分析

### 问题1：设备使用记录

**v3.0 方案**：记录设备使用情况（关联项目、使用时长、折旧成本）

**v4.0 方案**：**不记录设备使用记录**

**理由**：
- 用户不会愿意在系统里记录
- 自动关联的数据不准
- 自然折旧法下，设备成本按时间平均分摊，不需要追踪单次使用

**影响**：
- 删除 `equipment_usage` 表的使用场景
- 设备成本 = 设备年折旧额 / 年切片数量（按设备类型平均）
- 简化设备管理模块

---

### 问题2：标准工时粒度

**v3.0 方案**：按"项目类型 + 步骤"配置（所有IHC抗体共享同一工时）

**v4.0 方案**：**按"项目类型 + 步骤"配置（保持不变）**

**理由**：
- 用户选择A，基于最小成本法
- 如果不同检测项目工时差异大，可以通过调整"项目类型"的粒度来解决（如：IHC-简单、IHC-复杂）
- 但初期不需要细化到检测项目级别

**影响**：
- 保持现有 `standard_labor_times` 表的粒度
- 如果未来需要细化，可以增加项目子类型

---

### 问题3：季度调整审批

**v3.0 方案**：单人审核（财务人员确认即可）

**v4.0 方案**：**单人审核（保持不变）**

**理由**：
- 用户选择A，简化流程
- 第一阶段不需要多级审批
- 后续根据实际需求再增加

**影响**：
- 简化季度调整流程
- 删除多级审批的需求

---

### 问题4：成本预览缓存

**v3.0 方案**：BOM变更时自动更新缓存 + 手动刷新按钮

**v4.0 方案**：**混合策略（自动标记过期 + 提示用户手动刷新）**

**理由**：
- 用户选择C
- 自动标记过期保证数据准确性
- 提示用户手动刷新，让用户控制更新时机

**影响**：
- 物料价格变化时，自动标记相关BOM缓存为"可能过期"
- 用户查看成本预览时，如果缓存过期，显示提示"数据可能需要更新，建议刷新"
- 提供"刷新成本"按钮

---

### 问题5：物料级下钻字段

**v3.0 方案**：基础字段 + 供应商信息 + 物料单价

**v4.0 方案**：**仅基础字段**

**理由**：
- 用户选择A，最小必要字段
- 供应商信息和物料单价是敏感数据，增加权限控制的复杂度
- 物料级下钻的核心目的是分析成本偏差，基础字段已足够

**影响**：
- 物料级下钻字段：物料名称、规格、单位、实际用量、标准用量、偏差率、金额
- 不显示供应商信息和物料单价
- 简化权限控制

---

### 问题6：成本导出

**v3.0 方案**：支持Excel和PDF导出

**v4.0 方案**：**同时支持Excel和PDF导出（保持不变）**

**理由**：
- 用户选择C，灵活性
- Excel适合数据分析，PDF适合汇报

**影响**：
- 无变更

---

### 问题7：成本差异分析

**v3.0 方案**：支持两种对比（实际vs标准、本月vs上月）

**v4.0 方案**：**同时支持两种对比（保持不变）**

**理由**：
- 用户选择C，全面性
- 两种对比各有用途：实际vs标准找偏差，本月vs上月找趋势

**影响**：
- 无变更

## 五、v4.0 设备成本计算逻辑（核心变更）

### v3.0 方案（已删除）
```
设备成本 = Σ(单台设备使用时长 × 该设备折旧费率)
需要：设备使用记录（关联项目+使用时长）
问题：用户不会记录，自动关联不准
```

### v4.0 方案（推荐）
```
设备成本 = 设备类型年折旧额 / 年切片数量 × 该BOM切片数
不需要：设备使用记录
优势：数据来源可靠（设备台账+切片统计），无需用户录入
```

### 计算示例
```
假设：
- 全自动染色机（设备类型）有 2 台
- 设备1：购置价 50万，折旧年限 10年，年折旧 5万
- 设备2：购置价 30万，折旧年限 8年，年折旧 3.75万
- 该类型年折旧总额 = 5万 + 3.75万 = 8.75万
- 年切片数量 = 10000 张
- 设备类型平均折旧成本 = 8.75万 / 10000 = 8.75 元/张

某BOM需要染色机使用 30 分钟：
- 假设染色机每天工作 8 小时，年工作 250 天 = 120000 分钟
- 该BOM设备成本 = 8.75 × (30 / 120000) = 0.002 元（几乎可忽略）

或者简化为：
- 该BOM设备成本 = 设备类型平均折旧成本 × 权重系数
- 权重系数 = 该步骤标准耗时 / 所有步骤标准耗时之和
```

## 六、v4.0 成本计算公式（完整版）

```
单切片全成本 = 直接材料成本 + 人工成本 + 设备成本 + 间接成本

其中：
1. 直接材料成本 = Σ(组件用量 × 单价 × (1 + 损耗率)) + 通用配额
2. 人工成本 = Σ(标准工时 × 费率)
3. 设备成本 = Σ(设备类型年折旧额 / 年切片数量 × 权重系数)
4. 间接成本 = Σ(成本中心月度费用 / 月度切片数量)
```

### 数据来源

| 成本项 | 数据来源 | 用户录入 |
|--------|---------|---------|
| 直接材料成本 | 物料单价（物料表）+ BOM用量 | BOM配置时录入用量 |
| 人工成本 | 标准工时（standard_labor_times表）+ 费率 | 配置时录入 |
| 设备成本 | 设备台账（equipment表）+ 切片统计 | 设备台账录入一次 |
| 间接成本 | 成本中心（indirect_cost_centers表）+ 月度费用 | 每月录入费用 |

### 用户录入工作量

| 数据项 | 录入频率 | 录入人 | 工作量 |
|--------|---------|--------|--------|
| 设备台账 | 新增设备时 | 设备管理员 | 低（一次性） |
| 标准工时配置 | 初始配置 | 管理员 | 低（一次性） |
| BOM配置 | 新增/修改BOM时 | BOM管理员 | 中（按需） |
| 月度间接费用 | 每月 | 财务人员 | 低（每月一次） |
