## 一、5 个待确认问题分析

### 问题 1：设备类型的定义粒度

**业务场景**：
- 用户在配置 BOM 时，需要指定"这个检测项目需要哪些设备参与"
- 用户在查看成本报表时，需要了解"设备成本占总成本的比例"
- 用户在管理设备时，需要满足 ISO 15189 / CAP 的设备档案要求

**用户需求分析**：

| 角色 | 需要什么粒度 | 为什么 |
|------|------------|--------|
| BOM 管理员 | 设备类型（一级） | 配置 BOM 时只关心"需要染色机"，不关心具体哪台 |
| 设备管理员 | 单台设备 | ISO 15189 要求每台设备有独立档案 |
| 财务人员 | 设备类型（一级） | 看成本时关心"染色机类的平均折旧成本" |
| 实验室主管 | 设备类型 + 品牌 | 采购决策时需要对比不同品牌的成本效益 |

**推荐方案：A — 一级分类 + 品牌型号字段**

理由：
1. BOM 配置的核心需求是"按类型配置"，一级分类已满足
2. 单台设备的详细信息（品牌、型号）在设备台账中已有，不需要在类型层级重复
3. 多级分类增加 BOM 配置的复杂度，但用户在配置 BOM 时很少需要区分"全自动 vs 半自动"
4. 如果未来需要多级分类，可以在一级分类基础上扩展，不破坏现有数据结构
5. 与 v2.0 的分层管理方案一致：台账层管单台，BOM 层管类型

**具体定义**：
- 设备类型为一级分类：切片机、染色机、显微镜、数字切片扫描仪、PCR 仪、离心机、其他
- 每个设备类型下可有多台设备（设备台账）
- BOM 配置时关联设备类型 + 使用时长
- 成本核算时按设备类型的平均折旧成本计算

### 问题 2：行业参考值的数据来源

**业务场景**：
- 用户首次配置标准工时，不知道合理的耗时应该是多少
- 用户希望参考行业标准，降低配置门槛
- 用户需要信任参考值的来源，才会采纳

**用户需求分析**：

| 角色 | 需要什么来源 | 信任度要求 | 更新频率 |
|------|------------|----------|---------|
| 实验室主管 | 行业标准 + 内部经验 | 高（需要权威来源） | 低（1-2 年） |
| 财务人员 | 供应商提供的费率 | 中（需要验证） | 中（年度） |
| 新用户 | 系统预设值 | 中（需要可调整） | 低 |

**推荐方案：D — 综合来源（供应商数据为主 + 行业标准验证 + 用户可调整）**

理由：
1. 供应商数据最直接：设备供应商提供设备的标准处理时间，试剂供应商提供标准用量
2. 行业标准（CAP、ISO 15189）作为验证基准，增强用户信任
3. 用户可调整是必须的：每个实验室的实际条件不同，参考值必须可修改
4. 内部测定成本高、周期长，不适合作为初始数据来源
5. 系统已有预置标准工时数据（8 个步骤），可以作为默认参考值

**具体定义**：
- 参考值来源标记：`供应商提供` / `行业标准` / `系统预设`
- 每个参考值显示来源标签，增强用户信任
- 用户可一键采纳参考值，也可手动调整
- 参考值更新频率：年度审查（由管理员触发）
- 参考值与用户配置值分离：参考值只读，用户配置值可修改

### 问题 3：季度调整的触发条件

**业务场景**：
- 财务人员在季度末需要结算间接费用的实际发生额
- 月度预提可能与实际发生额有偏差，需要季度调整
- 调整可能涉及审批流程

**用户需求分析**：

| 角色 | 期望的流程 | 及时性要求 | 控制权要求 |
|------|----------|----------|----------|
| 财务人员 | 自动计算 + 手动确认 | 高（季度末及时调整） | 高（需要审核确认） |
| 实验室主管 | 查看调整结果 | 中（不影响日常操作） | 低（不需要干预） |
| 管理者 | 审批调整 | 中（月度报表前完成） | 高（需要审批权） |

**推荐方案：C — 混合触发（自动计算 + 手动确认）**

理由：
1. 自动计算减少财务人员的工作量：系统在季度末自动计算调整金额
2. 手动确认保证控制权：财务人员需要审核确认后才生效
3. 支持手动提前：如果财务人员需要提前调整，可以手动触发
4. 符合实际财务流程：季度结算需要人工审核，不能完全自动化
5. 与 v2.0 的"月度预提 + 季度调整"方案一致

**具体定义**：
- 系统在每季度末（3/6/9/12 月最后一天）自动生成调整建议
- 调整建议包含：预提金额、实际发生额、调整金额、调整原因
- 财务人员审核确认后，调整生效
- 支持手动触发：财务人员可随时手动触发季度调整
- 调整记录可追溯：记录调整时间、调整人、调整金额

### 问题 4：成本预览的更新频率

**业务场景**：
- 用户修改 BOM 后，希望立即看到成本变化
- 用户查看成本预览时，需要准确的成本数据
- 物料价格变化时，成本预览应该反映最新价格

**用户需求分析**：

| 角色 | 准确性要求 | 性能要求 | 期望的行为 |
|------|----------|---------|----------|
| BOM 管理员 | 高（修改后立即看到） | 中（可以等 1-2 秒） | 修改 BOM 后自动更新 |
| 财务人员 | 高（需要准确数据） | 高（报表加载要快） | 查看时显示最新数据 |
| 实验室主管 | 中（大致准确即可） | 高（快速查看） | 不需要频繁刷新 |

**推荐方案：B — 缓存更新（BOM 变更时更新缓存 + 手动刷新按钮）**

理由：
1. BOM 变更时自动更新缓存：用户修改 BOM 后立即看到成本变化，满足 BOM 管理员的需求
2. 缓存提高性能：报表页面加载时不需要实时计算，满足财务人员的性能需求
3. 手动刷新按钮：物料价格变化时，用户可以手动刷新所有 BOM 的成本预览
4. 兼顾准确性和性能：自动更新保证 BOM 修改时的准确性，缓存保证报表查看时的性能
5. 与现有实现一致：系统已有 `slide_cost_snapshots` 表，支持成本快照

**具体定义**：
- BOM 修改时自动重新计算该 BOM 的成本预览（异步，不阻塞页面）
- 报表页面显示缓存数据，标注"数据更新时间"
- 提供"刷新成本"按钮，手动触发全量重新计算
- 物料价格变化时，系统提示"成本预览可能需要更新"
- 成本预览显示"基于 BOM 版本 vX.X 计算"，让用户知道数据来源

### 问题 5：成本下钻的权限控制

**业务场景**：
- 财务人员需要查看详细的成本构成，分析成本异常
- 实验室主管需要了解项目成本，但不需要看到物料级别的敏感数据
- 采购人员需要了解物料成本，但不需要看到人工成本
- 管理者需要查看汇总数据，但不需要操作权限

**用户需求分析**：

| 角色 | 需要什么数据 | 分析需求 | 导出需求 |
|------|------------|---------|---------|
| 财务人员 | 全部数据 | 成本异常分析 | 导出完整报表 |
| 实验室主管 | 项目 + 样本级 | 项目成本优化 | 导出项目报表 |
| 采购人员 | 物料级 | 供应商成本对比 | 导出物料报表 |
| 管理者 | 项目级汇总 | 盈利性分析 | 导出汇总报表 |
| 技术员 | 项目级只读 | 了解项目成本 | 不需要导出 |

**推荐方案：C — 角色权限（不同角色有不同下钻权限）**

理由：
1. 角色权限最符合业务需求：不同角色需要不同的数据粒度
2. 与现有权限系统一致：系统已有 6 种角色，权限控制基于角色
3. 安全性最高：敏感数据（如物料单价、人工费率）只对授权角色可见
4. 用户体验好：用户只看到自己需要的数据，不会被无关信息干扰
5. 实现简单：基于现有角色权限系统扩展，不需要新的权限模型

**具体定义**：

| 下钻层级 | 可访问角色 | 说明 |
|---------|----------|------|
| 项目级汇总 | 所有角色 | 所有有成本报表权限的用户都能查看 |
| 样本级下钻 | 财务人员、实验室主管、admin | 需要分析样本级成本异常 |
| 物料级下钻 | 财务人员、采购人员、admin | 需要分析物料成本和供应商对比 |
| 成本导出 | 财务人员、admin | 导出完整成本报表 |
| 成本配置 | 财务人员、admin | 修改成本参数（费率、分摊规则等） |

## 三、版本对比（v2.0 vs v3.0）

| 维度 | v2.0 | v3.0 | 变更说明 |
|------|------|------|---------|
| **设备类型粒度** | 待确认（一级 vs 多级） | 一级分类 + 品牌型号字段 | 明确为一级分类，简化 BOM 配置 |

| **季度调整触发** | 待确认 | 混合触发（自动计算 + 手动确认） | 兼顾效率和控制权 |
| **成本预览更新** | 待确认 | 缓存更新（BOM 变更时更新 + 手动刷新） | 兼顾准确性和性能 |
| **成本下钻权限** | 待确认 | 角色权限（分层下钻） | 与现有权限系统一致 |
| **参考值来源标签** | 无 | 增加来源标签（供应商/行业/系统） | 新增功能 |
| **季度调整审核** | 无 | 增加审核确认流程 | 新增功能 |
| **成本预览刷新** | 无 | 增加手动刷新按钮 | 新增功能 |
