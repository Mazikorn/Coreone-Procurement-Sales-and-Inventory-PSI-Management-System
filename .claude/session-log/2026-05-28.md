### 网络错误修复（后续会话补充）

**问题**：部署后前端请求 `localhost:3001/api/v1` 导致 `ERR_CONNECTION_REFUSED`。

**根因**：`request.ts` 使用 `import.meta.env.VITE_API_BASE_URL || 'http://localhost:3001/api/v1'`，Windows 本地 build 时环境变量被解析为文件路径 `D:/Git/api/v1`，回退到 localhost。

**修复**：
- `request.ts`：硬编码 `baseURL: '/api/v1'`
- `vite.config.ts`：dev server 添加 `proxy: { '/api/v1': { target: 'http://localhost:3001', changeOrigin: true } }`
