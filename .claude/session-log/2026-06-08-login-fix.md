# 2026-06-08 E2E 登录超时根因修复

## 会话目标
诊断并修复 E2E 测试中 329 个失败用例的根因。

## 根因分析

### 发现过程
1. 探索 E2E 测试结构：73 个 spec 文件，329 个失败目录
2. 分析 error-context.md：所有失败都是 `waitForURL` 超时
3. 检查后端 auth.ts：发现 `express-rate-limit` (5次/分钟/IP) + 账户锁定 (5次失败/15分钟)
4. 确认：E2E 测试共享 127.0.0.1，速率限制被触发

### 根因
- **主因**: `express-rate-limit` 限制 5 次/分钟/IP，E2E 测试快速连续登录触发
- **次因**: 账户锁定机制（5 次失败/15 分钟）进一步加剧
- **表现**: API 返回 429，前端不导航，`waitForURL` 超时

## 修复方案

### 1. 后端修复 (`后端代码/server/src/routes/auth.ts`)
```typescript
// 速率限制：本地 IP 跳过
const loginLimiter = rateLimit({
  // ...
  skip: (req) => {
    const ip = req.ip || req.socket.remoteAddress || ''
    return ip === '127.0.0.1' || ip === '::1' || ip === '::ffff:127.0.0.1'
  },
})

// 账户锁定：本地 IP 跳过
const isLocal = ip === '127.0.0.1' || ip === '::1' || ip === '::ffff:127.0.0.1'
if (!isLocal) {
  // 检查账户锁定
}
```

### 2. 前端测试修复 (`前端代码/e2e/auth.spec.ts`)
```typescript
// beforeEach 精简
test.beforeEach(async ({ page }) => {
  await page.goto(`${FE_BASE}/login`, { waitUntil: 'domcontentloaded' })
  await page.evaluate(() => { localStorage.clear(); sessionStorage.clear() })
})

// loginAs 修复
async function loginAs(page: Page, role: RoleKey) {
  await page.goto(`${FE_BASE}/login`, { waitUntil: 'domcontentloaded' })
  await page.evaluate(() => { localStorage.clear(); sessionStorage.clear() })
  await page.goto(`${FE_BASE}/login`, { waitUntil: 'domcontentloaded' })
  // ...
  await page.waitForURL(`${FE_BASE}/`, { timeout: 15000, waitUntil: 'domcontentloaded' })
}
```

### 3. 共享 Helper 修复 (`前端代码/e2e/scenarios/shared/auth.ts`)
同步应用 `waitUntil: 'domcontentloaded'` 修复。
