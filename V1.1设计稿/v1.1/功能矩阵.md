## 一、抽象功能点定义

每个"列表管理型"页面通常由以下功能模块组成：

| # | 功能点 | 定义 | 判定标准 |
|:-:|:---|:---|:---|
| F01 | **统计卡片** | 页面顶部数据概览卡片，可点击筛选 | 有卡片 + 数字来自API |
| F02 | **快捷操作栏** | 主按钮区（新增/导入/导出/打印等） | 有按钮 + 点击有响应 |
| F03 | **快速筛选标签** |  pill 式标签（全部/今日/本周/本月等） | 有标签 + 切换生效 |
| F04 | **筛选栏** | 搜索框 + 下拉 + 日期范围等组合筛选 | 有控件 + 筛选生效 |
| F05 | **批量操作栏** | 选中行后顶部浮现的操作条 | 有选中态 + 批量操作可用 |
| F06 | **表格** | 数据列表展示 | 有表格 + 数据来自API |
| F07 | **分页** | 翻页控件 | 有分页 + 切换生效 |
| F08 | **新增弹窗** | 创建记录表单 | 有弹窗 + 提交后创建 |
| F09 | **编辑弹窗** | 修改记录表单 | 有弹窗 + 提交后更新 |
| F10 | **详情弹窗** | 查看单条记录完整信息 | 有弹窗 + 数据完整 |
| F11 | **删除确认** | 删除前的二次确认 | 有确认 + 删除生效 |
| F12 | **导出** | 数据导出为 Excel/CSV | 点击导出 + 文件可下载 |
| F13 | **打印** | 打印预览/直接打印 | 有预览 + 打印内容正确 |
| F14 | **扫码** | 扫码枪/摄像头识别条码 | 有扫码入口 + 能识别真实条码 |
| F15 | **批量导入** | Excel/CSV 批量导入 | 有导入入口 + 能解析真实文件 |
| F16 | **表格排序** | 点击表头按列排序 | 有排序交互 + 结果正确 |
| F17 | **URL筛选同步** | 筛选条件同步到URL，刷新不丢失 | URL变化 + 刷新后筛选保留 |
| F18 | **后端分页** | 分页由后端处理，非前端slice | 分页参数传给后端 + 后端返回total |
| F19 | **表单校验** | 必填/格式/范围等前端校验 | 有校验 + 错误提示明确 |
| F20 | **空状态** | 无数据时的友好提示 | 有提示 + 有引导操作 |
| F21 | **Loading状态** | 数据加载中的反馈 | 有loading + 不闪屏 |

## 四、跨页面共性问题（可批量修复）

### 4.1 所有列表页共有的问题

| 问题 | 影响页面 | 修复方式 |
|:---|:---|:---|
| **前端分页** | 入库、出库、库存、物料、供应商、BOM、预警、用户、日志 | 统一改为后端分页（1个API规范 + 各页面适配） |
| **无URL筛选同步** | 所有列表页 | 封装 `useUrlParams` hook，各页面接入 |
| **无表格排序** | 所有列表页 | 后端支持 `sortField`/`sortOrder`，表头加点击事件 |
| **导出为模拟/缺失** | 入库、物料、供应商、库位、项目等 | 统一实现导出工具函数（基于当前筛选结果） |
| **删除用原生confirm** | 入库、出库、物料、供应商、用户等 | 统一替换为自定义确认弹窗 |

### 4.2 特定功能点的问题

| 功能点 | 问题 | 影响页面 |
|:---|:---|:---|
| **扫码** | 仅入库有入口，但为模拟数据 | 入库 |
| **批量导入** | 仅入库/BOM有入口，但为模拟 | 入库、BOM |
| **打印** | 仅入库/出库有入口，操作人固定 | 入库、出库 |
| **编辑** | 出库无编辑弹窗 | 出库 |

## 六、确认后的执行计划

一旦确认，执行顺序建议：

```
Step 1: 批量修复共性问题（1次修改覆盖所有页面）
    ├─ 封装 useUrlParams hook → 所有列表页接入
    ├─ 封装后端分页 hook → 所有列表页接入
    ├─ 封装表格排序 hook → 所有列表页接入
    └─ 封装导出工具函数 → 所有列表页接入

Step 2: 批量修复特定功能（1次修改覆盖所有涉及页面）
    ├─ 自定义删除确认弹窗组件 → 替换所有原生confirm
    ├─ 扫码功能实现 → 接入真实条码识别
    ├─ 批量导入功能实现 → 接入真实文件解析
    └─ 打印功能完善 → 动态获取操作人

Step 3: 单页面问题修复
    └─ 按页面逐个修复演示数据/空壳逻辑

Step 4: 缺失页面开发
    └─ 退库 → 报废 → 调拨 → 采购订单
```
