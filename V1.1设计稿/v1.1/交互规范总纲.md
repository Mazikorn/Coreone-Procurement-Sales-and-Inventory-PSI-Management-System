# COREONE 交互规范总纲

> **目的**: 定义所有场景的统一交互标准，作为"可用"判定的唯一依据。  
> **原则**: 一个场景只有满足本规范的全部条目，才能标记为 ✅ 可用。

---

## 一、规范模板

每个场景的交互规范包含以下章节：

```
## 场景编号: 场景名称

### 1. 前置条件
### 2. 交互步骤（精确到每个动作和反馈）
### 3. 数据校验规则
### 4. API 调用定义
### 5. 状态变化
### 6. 错误处理
### 7. 验收标准（ checklist ）
```

---

## 二、共性交互标准（所有页面适用）

### 2.1 页面加载

| 检查项 | 标准 | 不合格示例 |
|:---|:---|:---|
| Loading 状态 | 数据加载时显示 loading，加载完成后隐藏 | 白屏或闪烁 |
| 空状态 | 无数据时显示友好提示+引导操作 | 空白表格或仅显示"暂无数据" |
| 首次加载 | 自动获取数据，无需用户手动点击"查询" | 需要点击"查询"才加载 |
| 错误提示 | 加载失败时显示错误信息和重试按钮 | 控制台报错但页面白屏 |

### 2.2 筛选与查询

| 检查项 | 标准 | 不合格示例 |
|:---|:---|:---|
| URL 同步 | 筛选条件同步到 URL query 参数 | 刷新后筛选丢失 |
| 实时筛选 | 下拉/onChange 实时生效，无需点击"查询" | 必须点击"查询"才生效 |
| 查询按钮 | 如存在查询按钮，点击后触发筛选 | 查询按钮为空函数 |
| 重置按钮 | 清空所有筛选条件，URL 同步清空 | 仅清空表单但 URL 不变 |
| 筛选联动 | 切换快速筛选时重置高级筛选，避免冲突 | 快速筛选和高级筛选同时生效导致无结果 |

### 2.3 分页

| 检查项 | 标准 | 不合格示例 |
|:---|:---|:---|
| 后端分页 | 分页参数传给后端，后端返回 total | 前端 `data.slice` 假分页 |
| 页码显示 | 显示"共 X 条，第 Y/Z 页" | 仅显示页码按钮 |
| 页码范围 | 最多显示 5-7 个页码，超出显示省略号 | 显示所有页码（100页就100个按钮） |
| 边界处理 | 上一页/下一页在边界时 disabled | 点击后无反应或报错 |
| 筛选后重置 | 任何筛选变化时重置到第 1 页 | 筛选后还在第 5 页导致空白 |

### 2.4 表格

| 检查项 | 标准 | 不合格示例 |
|:---|:---|:---|
| 表头排序 | 点击表头可按该列排序（升序/降序/取消） | 无排序功能 |
| 行选择 | checkbox 选中，选中行高亮 | 选中后无视觉反馈 |
| 全选 | 表头 checkbox 全选当前页 | 全选后翻页仍显示全选状态 |
| 批量操作栏 | 选中 ≥1 项时顶部显示操作栏，0 项时隐藏 | 始终显示或从不显示 |
| 空状态 | 无数据时显示插画+文字+引导按钮 | 仅显示"暂无数据" |

### 2.5 删除操作

| 检查项 | 标准 | 不合格示例 |
|:---|:---|:---|
| 二次确认 | 自定义弹窗（非原生 `confirm`） | 原生 `confirm()` |
| 影响展示 | 弹窗展示删除影响（如"该批次已有出库记录"） | 仅显示"确定删除？" |
| 预检查 | 删除前调 `checkDeletable` API | 删除后才发现不可删 |
| 级联提示 | 如有级联影响，列出具体影响项 | 直接删除导致数据不一致 |
| 成功反馈 | 删除成功 toast + 列表刷新 | 无反馈或页面不刷新 |

### 2.6 导出

| 检查项 | 标准 | 不合格示例 |
|:---|:---|:---|
| 真实文件 | 生成并下载真实文件（CSV/Excel） | 仅 toast"正在导出" |
| 文件名 | 包含日期和内容的文件名 | 固定文件名 |
| 内容完整 | 包含当前筛选条件下的全部数据 | 仅导出当前页或固定数据 |
| 编码正确 | 中文不乱码 | CSV 中文乱码 |

### 2.7 打印

| 检查项 | 标准 | 不合格示例 |
|:---|:---|:---|
| 操作人动态 | 打印单显示当前登录用户 | 固定"张医生" |
| 时间动态 | 显示当前系统时间 | 固定时间 |
| 内容完整 | 包含完整单据信息 | 内容缺失 |
| 打印样式 | 调用 `window.print()` 前加载打印样式 | 打印的是网页而非报表 |

### 2.8 表单弹窗

| 检查项 | 标准 | 不合格示例 |
|:---|:---|:---|
| 字段绑定 | 所有输入字段绑定 state | 使用 `defaultValue` |
| 必填校验 | 必填字段未填时阻止提交并提示 | 提交后后端返回 400 |
| 格式校验 | 邮箱/电话/日期等格式校验 | 任意字符串都通过 |
| 范围校验 | 数量/金额等数值范围校验 | 负数或超大数都通过 |
| 联动校验 | 关联字段联动校验（如采购订单数量≤剩余量） | 超量提交后后端报错 |
| 提交防重 | 提交中按钮 disabled + loading | 快速双击提交两次 |
| 错误提示 | 提交失败时显示具体错误（非"操作失败"） | toast.error("操作失败") |
| 成功反馈 | 提交成功 toast + 列表刷新 + 弹窗关闭 | 成功但弹窗未关 |

## 三、业务交互标准

### 3.1 入库标准

| 检查项 | 标准 | 不合格示例 |
|:---|:---|:---|
| 采购订单筛选 | 下拉显示 `pending` 和 `partial` 状态的订单 | 仅显示 `pending` |
| 剩余量提示 | 选择订单后数量旁显示"剩余待入: X" | 不显示剩余量 |
| 数量上限 | 输入数量超过 `remainingQty` 时阻止并提示 | 超量提交后后端 400 |
| 自动填充 | 选择订单后自动填耗材/供应商/单价 | 不自动填充 |
| 分批入库 | 同一张订单可分多次入库，每次更新 `received_qty` | 只能入库一次 |
| 状态流转 | `pending` → `partial` → `completed` | 状态不更新 |

### 3.2 出库标准

| 检查项 | 标准 | 不合格示例 |
|:---|:---|:---|
| 库存校验 | 出库数量 ≤ 当前库存 | 超库存出库 |
| 批次选择 | 必须选择具体批次 | 不选批次直接出库 |
| BOM关联 | 项目领用可关联BOM快速选择物料 | 手动逐条添加 |

### 3.3 库存标准

| 检查项 | 标准 | 不合格示例 |
|:---|:---|:---|
| 实时库存 | 入库/出库/删除后库存实时更新 | 操作后库存未变 |
| 负库存阻止 | 出库时库存不足阻止提交 | 允许负库存 |
| 批次追踪 | 按批次追踪库存流转 | 不区分批次 |

## 四、技术交互标准

### 4.1 API 调用

| 检查项 | 标准 | 不合格示例 |
|:---|:---|:---|
| 参数完整 | 请求参数与 API 定义一致 | 缺少必要参数 |
| 错误处理 | 所有 API 调用有 try-catch | `catch` 块为空或仅 console |
| 超时处理 | API 超时有友好提示 | 无响应无提示 |
| 重试机制 | 关键操作失败后可重试 | 失败后只能刷新页面 |

### 4.2 状态管理

| 检查项 | 标准 | 不合格示例 |
|:---|:---|:---|
| 表单绑定 | 使用受控组件（`value` + `onChange`） | `defaultValue` |
| 弹窗关闭 | 关闭后清理表单 state | 再次打开显示上次数据 |
| 列表刷新 | 增删改后自动刷新列表 | 手动刷新才看到新数据 |

## 五、"可用"判定 Checklist

一个场景标记为 ✅ 可用，必须全部满足：

- [ ] **前端交互完整**: 表单绑定 state，有校验，提交有反馈
- [ ] **后端真实调用**: 调用真实 API，非 mock/模拟
- [ ] **数据真实**: 无硬编码，无模拟数据
- [ ] **错误处理完善**: 失败时有明确提示，不静默
- [ ] **符合共性标准**: 满足本规范第 2 章对应条目
- [ ] **符合业务标准**: 满足本规范第 3 章对应条目

---

## 六、"修复后可用"的定义

对于标记为 ⚠️ 或 ❌ 的场景，修复后标记为 ✅ 的条件：

```
场景当前状态: ⚠️ 或 ❌
修复动作: [具体修改内容]
验证方式: [如何验证修复成功]
修复后状态: ✅
```
