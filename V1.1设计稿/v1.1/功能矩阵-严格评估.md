# COREONE 功能矩阵 — 严格评估版

> **评估日期**: 2026-05-22  
> **评估标准**: 端到端可用（非仅UI渲染）  
> **原则**: 只要存在模拟数据、空壳逻辑、硬编码、前端分页，相关场景一律不标✅

---

## 严格验收标准

### ✅ 可用（必须同时满足）

1. 前端交互完整：表单绑定 state、有校验、提交有反馈
2. 后端 API 真实调用：非 mock、非固定返回
3. 数据真实：非硬编码、非模拟数据
4. 错误处理完善：失败时有明确提示，不静默失败
5. 分页后端处理：或数据量小到不影响体验
6. URL 同步：筛选刷新后不丢失
7. 删除二次确认：非原生 `confirm()`，展示具体影响

### ⚠️ 有缺陷（满足任一即标）

1. 前端分页（`data.slice`）— 数据量大时卡死
2. URL 不同步 — 刷新后筛选丢失
3. 无表格排序
4. 删除用原生 `confirm()`
5. 表单校验不完善（仅必填，无范围/格式校验）
6. 统计卡片有硬编码 fallback（如 `|| 156`）
7. 查询按钮为空函数（虽然 onChange 已生效）
8. 状态判断为演示逻辑（如 `quantity > 1000`）

### ❌ 缺失（满足任一即标）

1. 模拟数据（扫码固定返回、导入固定 50 条）
2. 空壳逻辑（弹窗打开但不调 API）
3. 硬编码计算（恢复库存 `+400`、操作人固定"张医生"）
4. 功能完全缺失（无页面、无 API）
5. Toast 模拟（导出时仅弹"正在导出"，无真实文件下载）
