## 统计卡片

### DB-01~04: 查看统计卡片

| 属性 | 内容 |
|:---|:---|
| **角色** | 全部角色 |
| **触发** | 进入仪表盘页面 |
| **当前状态** | ⚠️ 有缺陷 |

**交互步骤**:
1. 进入页面 → 调 `GET /dashboard/stats` 获取统计数据
2. 渲染4张统计卡片：
   - DB-01: 库存总量
   - DB-02: 本月入库
   - DB-03: 本月出库
   - DB-04: 预警数量
3. 卡片显示数字 + 环比变化百分比

**缺陷描述**:
- 数字来自后端 API，但未验证是否为真实业务数据（可能为 seed 初始化数据）
- 部分统计卡片可能存在硬编码 fallback（如 `|| 156` 等默认值）

**规范定义**:
- 数字必须来自后端真实业务统计 API
- 无数据时显示 `0` 或 `-`，不使用硬编码数字
- 环比计算基于真实上月/上周数据

**修复方案**:
- 移除所有 `|| 默认值` 的硬编码 fallback
- 确认 `GET /dashboard/stats` 返回的是真实业务聚合数据，非 seed 数据
- 或新增 `GET /dashboard/stats/real` 接口，从真实入库/出库/库存记录中实时计算

**验收标准**:
- [ ] 统计数字与数据库真实记录一致（手动抽查验证）
- [ ] 无硬编码 fallback
- [ ] 空数据时显示合理值（`0` 或 `-`）
- [ ] 环比变化计算正确

## 数据图表

### DB-09: 查看库存趋势图

| 属性 | 内容 |
|:---|:---|
| **角色** | 全部角色 |
| **触发** | 进入仪表盘页面 |
| **当前状态** | ⚠️ 有缺陷 |

**交互步骤**:
1. 进入页面 → 调 `GET /dashboard/trend` 获取近6个月库存趋势数据
2. 渲染折线图/面积图：
   - X轴：月份（近6个月）
   - Y轴：库存数量/金额
   - 系列：在库总量、入库量、出库量
3. 图表支持 hover 显示具体数值

**缺陷描述**:
- 图表展示正常，但数据未验证真实性（可能为 seed 数据或模拟数据）
- 趋势数据可能未从真实库存快照中计算

**规范定义**:
- 趋势数据必须从真实业务记录中按月聚合
- 近6个月定义：当前月往前推5个月，共6个月
- 每月数据点：月末库存快照 或 当月累计入库/出库

**修复方案**:
- 确认后端 `GET /dashboard/trend` 从真实 `inbound`/`outbound`/`inventory` 表中聚合
- 或新增按月统计表，定时任务更新

**验收标准**:
- [ ] 图表数据与真实业务记录一致（手动抽查某月数据）
- [ ] 近6个月范围正确
- [ ] Hover 显示精确数值
- [ ] 无数据月份显示为 `0` 而非插值

## 最近活动

### DB-10: 查看最近活动列表

| 属性 | 内容 |
|:---|:---|
| **角色** | 全部角色 |
| **触发** | 进入仪表盘页面 |
| **当前状态** | ⚠️ 有缺陷 |

**交互步骤**:
1. 进入页面 → 调 `GET /dashboard/activities` 获取最近活动
2. 渲染活动列表（通常显示5~10条）：
   - 活动类型：入库 / 出库 / 预警 / 盘点
   - 显示内容：操作描述、操作人、时间
   - 示例："张医生 入库了 抗体试剂A 50盒"
3. 点击单条活动 → 跳转到对应详情页

**缺陷描述**:
- 列表展示正常，但数据来源未验证（可能为 seed 数据）
- 活动记录可能未从真实操作日志中读取

**规范定义**:
- 活动数据来自真实操作日志表（`logs` 或 `activities`）
- 按时间倒序，取最近 N 条
- 显示格式：操作人 + 操作类型 + 对象 + 时间

**修复方案**:
- 确认 `GET /dashboard/activities` 从真实日志表中查询
- 或从 `inbound`/`outbound`/`alerts` 等表中按时间倒序取最近记录

**验收标准**:
- [ ] 列表数据与真实操作记录一致
- [ ] 按时间倒序排列
- [ ] 显示操作人、操作类型、对象、时间
- [ ] 点击单条可跳转到对应详情页
- [ ] 无活动时显示"暂无动态"空状态
