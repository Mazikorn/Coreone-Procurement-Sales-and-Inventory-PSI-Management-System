## 统计卡片

### INV-01~06: 查看统计卡片

| 属性 | 内容 |
|:---|:---|
| **角色** | 全部角色（部分卡片按角色权限显示） |
| **触发** | 进入页面 |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 进入页面 → 调 `GET /inventory/stats` 获取统计数据
2. 显示统计卡片：
   - INV-01 库存总量（全部角色）
   - INV-02 正常库存（全部角色）
   - INV-03 库存不足（warehouse_manager, procurement）
   - INV-04 临期预警（warehouse_manager, technician）
   - INV-05 已过期（warehouse_manager）
   - INV-06 缺货（procurement）
3. 点击卡片可筛选对应状态

**验收标准**:
- [ ] 数字来自后端 API
- [ ] 空数据时显示 `0` 或 `-`
- [ ] 点击卡片筛选生效

### INV-29: 批量报废

| 属性 | 内容 |
|:---|:---|
| **角色** | warehouse_manager |
| **触发** | 选中后点击"批量报废" |
| **当前状态** | ❌ 缺失 |

**缺陷**: 功能为模拟，未调真实 API

**规范定义**:
1. 选中多行 → 点击"批量报废"
2. 打开报废弹窗（size=md）
3. 弹窗显示：
   - 选中物料列表（名称/编码/批次/当前数量）
   - 报废原因下拉（过期/损坏/过期/其他）
   - 备注输入框
4. 点击"确认报废" → 调 `POST /inventory/scrap` → 成功后刷新

**API定义**:
```
POST /api/v1/inventory/scrap
Body: {
  items: [
    { materialId, batchId?, quantity, reason }
  ],
  remark?: string
}
```

**修复方案**:
- 后端新增 `POST /inventory/scrap` 接口（事务保护）
- 前端弹窗绑定表单 state
- 报废后库存扣减，生成出库记录（type=scrap）

**验收标准**:
- [ ] 弹窗显示选中物料列表
- [ ] 报废原因必填
- [ ] 提交后库存扣减
- [ ] 生成报废记录

## 分页

### INV-31~32: 分页操作

| 属性 | 内容 |
|:---|:---|
| **角色** | 全部角色 |
| **触发** | 点击页码/调整每页条数 |
| **当前状态** | ⚠️ 有缺陷 |

**缺陷**: 前端 `filteredData.slice`

**规范定义**（引用总纲 2.3）:
1. 分页参数传给后端：`GET /inventory?page=2&pageSize=20`
2. 后端返回 `{ list, pagination: { total, page, pageSize } }`
3. 切换页码时 URL 同步：`?page=2`
4. 调整 pageSize 时重置到第 1 页

**修复方案**:
- 后端 `GET /inventory` 已支持分页参数，但前端未传递
- 前端修改 `fetchData` 传递 `page`/`pageSize`
- 封装通用分页 hook

**验收标准**（引用总纲 2.3）:
- [ ] 分页参数传给后端
- [ ] 显示"共 X 条"
- [ ] URL 同步
- [ ] 调整 pageSize 重置分页
