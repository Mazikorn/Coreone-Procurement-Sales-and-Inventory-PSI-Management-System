### AL-13: 处理预警

| 属性 | 内容 |
|:---|:---|
| **角色** | wh |
| **触发** | 点击"处理" |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 点击"处理" → 打开 handle 弹窗（size=md）
2. 弹窗显示预警基本信息
3. 输入处理意见（textarea, 必填）
4. 选择处理结果（下拉：已补货 / 已调整 / 已确认 / 其他）
5. 点击"提交" → 校验 → 调 `PUT /alerts/:id/handle` → 成功后关闭弹窗 + 刷新列表

**API定义**:
```
PUT /api/v1/alerts/:id/handle
Body: { handleResult: string, handleNote: string }
Response: { success: true, data: {...} }
```

**验收标准**:
- [ ] 弹窗正常打开
- [ ] 处理意见必填
- [ ] 提交后状态变为"已处理"
- [ ] 列表刷新

## 分页

### AL-21: 翻页

| 属性 | 内容 |
|:---|:---|
| **角色** | wh, admin |
| **触发** | 点击页码 |
| **当前状态** | ⚠️ 有缺陷 |

**缺陷**: 前端 `filteredData.slice`

**规范定义**（引用总纲 2.3）:
1. 分页参数传给后端：`GET /alerts?page=2&pageSize=20`
2. 后端返回 `{ list, pagination: { total, page, pageSize } }`
3. 切换页码时 URL 同步：`?page=2`
4. 显示"共 X 条"
5. 筛选变化时重置到第 1 页

**修复方案**:
- 后端 `GET /alerts` 已支持分页参数，但前端未传递
- 前端修改 `fetchData` 传递 `page`/`pageSize`
- 封装通用分页 hook

**验收标准**（引用总纲 2.3）:
- [ ] 分页参数传给后端
- [ ] 显示"共 X 条"
- [ ] URL 同步
- [ ] 筛选后重置分页
