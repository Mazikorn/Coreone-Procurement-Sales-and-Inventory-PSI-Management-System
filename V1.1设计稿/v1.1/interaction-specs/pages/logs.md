## 统计卡片

### LOG-01~04: 查看统计卡片

| 属性 | 内容 |
|:---|:---|
| **角色** | admin |
| **触发** | 进入页面 |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 进入 `/logs` 页面 → 自动加载统计卡片数据
2. 卡片显示：日志总数 / 操作类型数 / 涉及模块数 / 活跃用户数
3. 数据来自 `GET /logs/stats` 或后端真实统计

**字段说明**:
- LOG-01 查看日志总数：显示系统累计日志条数
- LOG-02 查看操作类型数：显示不同操作类型的数量（如创建/更新/删除/查询）
- LOG-03 查看涉及模块数：显示有日志记录的模块数量
- LOG-04 查看活跃用户数：显示有操作记录的用户数量

**验收标准**:
- [ ] 数字来自后端 API，无硬编码 fallback
- [ ] 无数据时显示 `0` 或 `-`
- [ ] 加载失败时显示错误提示

## 筛选栏

### LOG-05~09: 筛选栏操作

| 属性 | 内容 |
|:---|:---|
| **角色** | admin |
| **触发** | 输入/选择筛选条件 |
| **当前状态** | ⚠️ 有缺陷 |

**缺陷汇总**:
- 全部筛选：URL 不同步
- 筛选生效，但分页为前端 `filteredData.slice`

**规范定义**（引用总纲 2.2）:
1. 日期范围筛选：选择起止日期后实时生效（或点击"查询"生效）
2. 操作类型筛选：onChange 实时生效（创建/更新/删除/查询等）
3. 模块筛选：onChange 实时生效（库存/入库/出库/用户等）
4. 关键词搜索：debounce 300ms，按用户名/操作内容搜索
5. "重置"按钮：清空所有筛选条件 + URL query
6. 筛选变化时重置到第 1 页

**修复方案**:
- 筛选条件同步到 URL：`?startDate=2026-01-01&endDate=2026-01-31&type=create&module=inventory&search=xxx`
- 筛选参数传给后端 `GET /logs?startDate=xxx&endDate=xxx&type=create`
- 封装通用筛选 hook，统一管理 URL 同步

**验收标准**（引用总纲 2.2）:
- [ ] URL 同步
- [ ] 筛选后重置分页
- [ ] 重置清空所有条件

## 导出日志

### LOG-10: 导出日志

| 属性 | 内容 |
|:---|:---|
| **角色** | admin |
| **触发** | 点击"导出"按钮 |
| **当前状态** | ⚠️ 有缺陷 |

**缺陷**: 导出功能待验证，可能仅为 toast 模拟

**规范定义**（引用总纲 2.6）:
1. 点击"导出" → 调 `POST /logs/export`
2. 参数：当前筛选条件（日期范围/操作类型/模块/关键词）
3. 后端生成 CSV/Excel → 返回文件流
4. 前端触发文件下载
5. 文件名格式：`logs_20260522_143052.csv`

**API定义**:
```
POST /api/v1/logs/export
Body: { startDate, endDate, type, module, search }
Response: Blob (CSV/Excel 文件流)
```

**修复方案**:
- 验证后端是否已实现 `POST /logs/export` 接口
- 如未实现：后端新增接口，支持按筛选条件导出
- 前端使用 `Blob` + `URL.createObjectURL` 触发下载

**验收标准**（引用总纲 2.6）:
- [ ] 下载真实文件
- [ ] 文件名包含日期
- [ ] 内容为当前筛选结果（非仅当前页）
- [ ] 中文不乱码

## 清理日志

### LOG-11: 清理历史日志

| 属性 | 内容 |
|:---|:---|
| **角色** | admin |
| **触发** | 点击"清理"按钮 |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 点击"清理" → 打开清理弹窗
2. 弹窗显示清理范围选项：
   - 清理 30 天前的日志
   - 清理 90 天前的日志
   - 清理 180 天前的日志
   - 清理全部日志
3. 选择范围 → 显示预估清理条数
4. 点击"确认清理" → 二次确认弹窗
5. 调 `DELETE /logs?beforeDate=xxx` → 成功 toast + 刷新列表

**验收标准**:
- [ ] 弹窗显示清理范围选项
- [ ] 显示预估清理条数
- [ ] 二次确认后才执行
- [ ] 清理成功后列表刷新
- [ ] 统计卡片数据更新

## 分页

### LOG-12: 分页操作

| 属性 | 内容 |
|:---|:---|
| **角色** | admin |
| **触发** | 点击页码/上一页/下一页 |
| **当前状态** | ⚠️ 有缺陷 |

**缺陷**: 前端 `filteredData.slice`

**规范定义**（引用总纲 2.3）:
1. 分页参数传给后端：`GET /logs?page=2&pageSize=20`
2. 后端返回 `{ list, pagination: { total, page, pageSize } }`
3. 切换页码时 URL 同步：`?page=2`
4. 筛选变化时重置到第 1 页
5. 显示"共 X 条，第 Y/Z 页"

**修复方案**:
- 后端 `GET /logs` 已支持分页参数，前端未传递
- 前端修改 `fetchData` 传递 `page`/`pageSize`
- 封装通用分页 hook

**验收标准**（引用总纲 2.3）:
- [ ] 分页参数传给后端
- [ ] 显示"共 X 条"
- [ ] URL 同步
- [ ] 筛选后重置分页
- [ ] 边界页码 disabled
