## 数据查看

### ROL-01~04: 查看角色列表

| 属性 | 内容 |
|:---|:---|
| **角色** | admin |
| **触发** | 进入页面 |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 进入 `/roles` 页面 → 自动加载角色列表
2. 表格显示：角色名称 / 角色编码 / 关联用户数 / 操作列
3. 数据来自 `GET /roles`

**字段说明**:
- ROL-01 查看角色列表：进入页面显示角色表格
- ROL-02 查看角色名称：表格显示名称列
- ROL-03 查看角色编码：表格显示编码列（如 `admin`, `warehouse_manager`）
- ROL-04 查看关联用户数：表格显示该角色下的用户数量

**验收标准**:
- [ ] 表格正确显示所有角色
- [ ] 关联用户数为真实统计
- [ ] 加载失败时显示错误提示

### ROL-06: 编辑角色权限

| 属性 | 内容 |
|:---|:---|
| **角色** | admin |
| **触发** | 点击"编辑" |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 点击"编辑" → 打开权限配置弹窗/抽屉
2. 弹窗显示所有可配置权限，当前角色已有权限为勾选状态
3. 勾选/取消勾选权限 → 调 `PUT /roles/:id/permissions` → 成功 toast + 刷新

**验收标准**:
- [ ] 弹窗正确打开，预填充当前权限
- [ ] 权限变更实时保存或批量保存
- [ ] 保存成功后列表刷新

---

### ROL-07: 删除角色

| 属性 | 内容 |
|:---|:---|
| **角色** | admin |
| **触发** | 点击"删除" |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 点击"删除" → 检查角色关联用户数
2. 打开自定义确认弹窗
3. 弹窗显示：角色名称、关联用户数
4. 若关联用户 > 0：显示"该角色下还有 X 个用户，请先转移用户"，删除按钮 disabled
5. 若关联用户 = 0：点击"确认删除" → 调 `DELETE /roles/:id` → 成功 toast + 刷新

**验收标准**（引用总纲 2.5）:
- [ ] 自定义确认弹窗（非原生 confirm）
- [ ] 展示删除影响（关联用户数）
- [ ] 有关联用户时禁用删除按钮
- [ ] 删除成功后列表刷新

### ROL-10: 保存权限配置

| 属性 | 内容 |
|:---|:---|
| **角色** | admin |
| **触发** | 点击"保存" |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 点击"保存" → 收集当前所有勾选权限
2. 调 `PUT /roles/:id/permissions` → Body: `{ permissions: [...] }`
3. 成功 toast"权限配置已更新"
4. 关闭弹窗，刷新列表

**API定义**:
```
PUT /api/v1/roles/:id/permissions
Body: { permissions: ["inventory:read", "inbound:create", ...] }
Response: { success: true, data: { id, name, permissions } }
```

**验收标准**（引用总纲 2.8）:
- [ ] 提交中按钮 disabled
- [ ] 成功 toast + 关闭弹窗 + 刷新列表
- [ ] 错误时显示具体错误信息
