## 筛选与导出

### CA-13: 选择时间范围

| 属性 | 内容 |
|:---|:---|
| **角色** | finance |
| **触发** | 选择日期范围 |
| **当前状态** | ⚠️ 有缺陷 |

**缺陷**: URL 不同步

**规范定义**:
1. 选择日期范围 → 图表和表格数据按范围刷新
2. 更新 URL：`?startDate=2026-01-01&endDate=2026-05-22`
3. 刷新页面后保留时间范围筛选
4. 时间范围影响所有 Tab 的数据

**修复方案**:
- 使用 `useSearchParams` 同步时间范围到 URL
- 页面加载时从 URL 读取初始时间范围
- 时间范围变化时更新 URL

**验收标准**（引用总纲 2.2）:
- [ ] URL 同步
- [ ] 刷新后保留筛选
- [ ] 时间范围格式统一（YYYY-MM-DD）

### CA-15: 导出成本报告

| 属性 | 内容 |
|:---|:---|
| **角色** | finance |
| **触发** | 点击"导出" |
| **当前状态** | ⚠️ 有缺陷 |

**缺陷**: 导出功能待验证，URL 不同步

**规范定义**（引用总纲 2.6）:
1. 点击"导出" → 打开 export 弹窗（size=md）
2. 弹窗选择：
   - 导出格式：Excel / CSV / PDF
   - 导出范围：当前 Tab 数据 / 全部 Tab 数据
   - 时间范围：默认当前筛选范围，可修改
3. 点击"确认导出" → 调 `POST /cost-analysis/export`
4. 参数：tab + 时间范围 + 数据来源 + 格式
5. 后端生成文件 → 返回文件流
6. 前端触发文件下载

**API定义**:
```
POST /api/v1/cost-analysis/export
Body: {
  tab: "project|material|common|supplier",
  format: "xlsx|csv|pdf",
  startDate: "2026-01-01",
  endDate: "2026-05-22",
  dataSource: "lis|manual|all"
}
Response: Blob (文件流)
```

**修复方案**:
- 验证当前实现是否真实下载文件
- 如为 toast 模拟，接入真实后端接口
- 使用 `useSearchParams` 同步筛选条件到 URL
- 前端使用 `Blob` + `URL.createObjectURL` 触发下载

**验收标准**（引用总纲 2.2、2.6）:
- [ ] URL 同步
- [ ] 下载真实文件
- [ ] 文件名包含日期和内容类型（如 `成本分析_检测项目_2026-05-22.xlsx`）
- [ ] 内容为当前筛选结果
- [ ] 中文不乱码
