**规范定义**（引用总纲 2.2）:

**修复方案**:
- 使用 `useSearchParams` 管理筛选状态
- 重置时调用 `setSearchParams({})`
- 分页参数同步重置

**验收标准**（引用总纲 2.2）:

## 分页

### PRO-21: 翻页

| 属性 | 内容 |
|:---|:---|
| **角色** | admin, technician, pathologist |
| **触发** | 点击页码 |
| **当前状态** | ⚠️ 有缺陷 |

**缺陷**: 前端 `filteredData.slice` 假分页

**规范定义**（引用总纲 2.3）:
1. 分页参数传给后端：`GET /projects?page=2&pageSize=20`
2. 后端返回 `{ list, pagination: { total, page, pageSize } }`
3. 切换页码时 URL 同步：`?page=2`
4. 筛选后重置到第 1 页

**修复方案**:
- 前端修改 `fetchData` 传递 `page`/`pageSize`
- 后端 `GET /projects` 已支持分页参数，前端未传递
- 封装通用分页 hook

**验收标准**（引用总纲 2.3）:
- [ ] 分页参数传给后端
- [ ] 显示"共 X 条"
- [ ] URL 同步
- [ ] 筛选后重置分页
