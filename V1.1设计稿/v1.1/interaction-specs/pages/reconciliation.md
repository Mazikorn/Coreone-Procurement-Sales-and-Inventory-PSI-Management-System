### REC-07: 查看差异

| 属性 | 内容 |
|:---|:---|
| **角色** | finance, tech |
| **触发** | 展开项目行 |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 点击项目行展开按钮 → 展开子表格
2. 显示差异量 = 实际消耗 - 理论消耗
3. 显示差异率 = (差异量 / 理论消耗) × 100%
4. 差异率为正（超耗）标红，为负（节约）标绿

**验收标准**:
- [ ] 差异计算正确
- [ ] 正负差异颜色区分正确
- [ ] 理论消耗为 0 时不计算差异率，显示 "N/A"

## 操作

### REC-09: 修正 BOM 用量

| 属性 | 内容 |
|:---|:---|
| **角色** | tech |
| **触发** | 点击"修正 BOM" |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 点击"修正 BOM" → 打开 fix 弹窗（size=md）
2. 弹窗显示：
   - 项目名称、物料名称
   - 当前 BOM 用量
   - 输入框：新用量（type=number, step=0.01, min=0）
   - 输入框：修正原因（textarea, 必填）
3. 点击"保存" → 校验 → 调 `PUT /bom/:id/usage` → 成功后关闭弹窗 + 刷新列表
4. 自动在修正日志中新增一条记录

**验收标准**:
- [ ] 弹窗预填充当前用量
- [ ] 修正原因必填
- [ ] 新用量必须 ≥ 0
- [ ] 保存后列表刷新，修正日志同步更新

### REC-10: 导入 LIS 数据

| 属性 | 内容 |
|:---|:---|
| **角色** | finance, admin |
| **触发** | 点击"导入 LIS" |
| **当前状态** | ⚠️ 有缺陷 |

**缺陷**: 导入功能待验证，不确定是否为真实 API 调用还是模拟数据

**规范定义**:
1. 点击"导入 LIS" → 打开 import 弹窗（size=lg）
2. 弹窗显示：
   - 文件上传区域（支持 .xlsx / .csv / .txt）
   - LIS 数据格式说明（病理号、检测项目、检测时间）
   - 下载模板链接
3. 上传文件 → 前端解析 → 逐条校验
4. 校验通过 → 显示预览（成功 X 条，失败 Y 条）
5. 点击"开始导入" → 调 `POST /reconciliation/import-lis` → 成功后关闭弹窗 + 刷新列表

**校验规则**:
- 病理号非空
- 检测项目编码必须存在
- 检测时间格式正确

**API定义**:
```
POST /api/v1/reconciliation/import-lis
Body: FormData (file)
Response: { success: true, imported: 100, failed: 5, errors: [...] }
```

**修复方案**:
- 验证当前实现是否调用了真实 API
- 如为模拟，接入真实后端接口
- 后端需支持 LIS 数据解析和病理号-项目关联

**验收标准**:
- [ ] 能上传真实 LIS 数据文件
- [ ] 解析后显示校验结果
- [ ] 导入成功后对账数据更新
- [ ] 导入失败时显示具体错误行

### REC-11: 导出对账报表

| 属性 | 内容 |
|:---|:---|
| **角色** | finance |
| **触发** | 点击"导出" |
| **当前状态** | ⚠️ 有缺陷 |

**缺陷**: 导出功能待验证，不确定是否生成真实文件下载

**规范定义**（引用总纲 2.6）:
1. 点击"导出" → 打开 export 弹窗（size=md）
2. 弹窗选择导出格式：Excel / CSV
3. 弹窗选择导出范围：当前 Tab 全部数据 / 当前筛选结果
4. 点击"确认导出" → 调 `POST /reconciliation/export`
5. 参数：当前 Tab 类型 + 筛选条件 + 格式
6. 后端生成文件 → 返回文件流
7. 前端触发文件下载

**API定义**:
```
POST /api/v1/reconciliation/export
Body: { tab: "project|material|case", format: "xlsx|csv", filters: {...} }
Response: Blob (文件流)
```

**修复方案**:
- 验证当前实现是否真实下载文件
- 如为 toast 模拟，接入真实后端接口
- 前端使用 `Blob` + `URL.createObjectURL` 触发下载

**验收标准**（引用总纲 2.6）:
- [ ] 下载真实文件
- [ ] 文件名包含日期和内容类型（如 `消耗对账_按项目_2026-05-22.xlsx`）
- [ ] 内容为当前筛选结果
- [ ] 中文不乱码
