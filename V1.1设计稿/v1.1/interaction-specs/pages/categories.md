## 数据查看

### CAT-01: 查看分类树

| 属性 | 内容 |
|:---|:---|
| **角色** | 全部角色 |
| **触发** | 进入页面 |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 进入 `/categories` 页面 → 自动加载分类数据
2. 左侧显示树形结构，根节点为"全部分类"
3. 每个节点显示分类名称和编码

**验收标准**:
- [ ] 树形结构正确渲染
- [ ] 数据来自后端 API，非硬编码
- [ ] 加载失败时显示错误提示

## 分类管理

### CAT-06: 新建一级分类

| 属性 | 内容 |
|:---|:---|
| **角色** | admin |
| **触发** | 右键根节点 / 点击"新建一级分类"按钮 |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 右键根节点或点击按钮 → 打开 create 弹窗（size=md）
2. 表单字段：分类名称（必填）、分类编码（必填）、备注（可选）
3. 点击"保存" → 前端校验 → 调 `POST /categories` → 成功关闭弹窗 + 刷新树

**验收标准**:
- [ ] 弹窗正确打开
- [ ] 名称和编码必填校验
- [ ] 编码唯一性校验（后端返回 409 时提示）
- [ ] 成功后树自动刷新，新分类出现在根节点下

---

### CAT-07: 新建子分类

| 属性 | 内容 |
|:---|:---|
| **角色** | admin |
| **触发** | 右键父节点 → 选择"新建子分类" |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 右键父分类节点 → 上下文菜单显示"新建子分类"
2. 点击后打开 create 弹窗，父分类已预填充且不可修改
3. 填写名称、编码、备注 → 调 `POST /categories`（含 `parentId`）
4. 成功后关闭弹窗 + 刷新树 + 自动展开父节点

**验收标准**:
- [ ] 右键菜单正确显示
- [ ] 父分类预填充且只读
- [ ] 成功后父节点自动展开显示新子分类

---

### CAT-08: 编辑分类

| 属性 | 内容 |
|:---|:---|
| **角色** | admin |
| **触发** | 右键节点 → 选择"编辑" |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 右键分类节点 → 上下文菜单显示"编辑"
2. 点击后打开 edit 弹窗，预填充当前分类数据
3. 可修改：名称、编码、备注（父分类不可修改）

**验收标准**:
- [ ] 弹窗预填充当前数据
- [ ] 父分类字段只读
- [ ] 提交后树中节点名称即时更新

---

### CAT-09: 删除分类

| 属性 | 内容 |
|:---|:---|
| **角色** | admin |
| **触发** | 右键节点 → 选择"删除" |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 右键分类节点 → 上下文菜单显示"删除"
2. 点击后调 `GET /categories/:id/check-deletable`
3. 打开自定义确认弹窗，显示：
   - 分类名称、编码
   - 删除影响（如"该分类下有关联物料 5 个"）
   - 若不可删除：显示原因，"删除"按钮 disabled
4. 点击"确认删除" → 调 `DELETE /categories/:id` → 成功 toast + 刷新树

**验收标准**（引用总纲 2.5）:
- [ ] 自定义确认弹窗（非原生 confirm）
- [ ] 展示删除影响（关联物料数）
- [ ] 不可删除时禁用按钮
- [ ] 成功后树中节点移除
