## 快捷操作

### OUT-01: 新建出库

| 属性 | 内容 |
|:---|:---|
| **角色** | warehouse_manager |
| **触发** | 点击"出库登记"按钮 |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 点击按钮 → 打开 create 弹窗（size=xl）
2. 表单字段：出库类型/检测项目/物料明细/批次/数量/领用人/备注
3. 点击"确认出库" → 校验 → 调 `POST /outbound` → 成功关闭弹窗+刷新列表

**验收标准**:
- [ ] 弹窗正确打开，无数据残留
- [ ] 必填字段（出库类型、物料、数量）未填时阻止提交
- [ ] 提交成功后 toast"出库成功"，列表刷新

### OUT-02: 打印记录

| 属性 | 内容 |
|:---|:---|
| **角色** | warehouse_manager |
| **触发** | 点击"打印记录"按钮 |
| **当前状态** | ❌ 缺失（基于推测） |

**缺陷**: 未扫描代码，但参照入库页面打印功能模式（IN-04/IN-27），大概率存在操作人固定为"张医生"或类似硬编码问题。

**规范定义**:
1. 点击按钮 → 打开 print 弹窗（size=lg）
2. 弹窗显示打印预览表格
3. 打印单包含：操作人（当前登录用户）、生成时间、出库记录列表
4. 点击"打印" → `window.print()`

**修复方案**:
- 从 JWT payload 或 localStorage 获取当前用户名
- 打印样式使用 `@media print` 隐藏不需要的元素

**验收标准**:
- [ ] 打印单显示真实操作人
- [ ] 打印单显示当前时间
- [ ] 打印内容格式正确

## 筛选栏

### OUT-12~17: 筛选栏操作

| 属性 | 内容 |
|:---|:---|
| **角色** | admin, warehouse_manager |
| **触发** | 输入/选择筛选条件 |
| **当前状态** | ⚠️ 有缺陷 |

**缺陷汇总**:
- 全部筛选：URL 不同步
- 前端分页导致大数据量时性能差

**规范定义**:
1. 搜索框实时搜索（debounce 300ms）
2. 下拉筛选 onChange 实时生效
3. 日期范围选择后需点击"查询"按钮生效（或 onChange 生效但按钮保留用于显式触发）
4. "查询"按钮：如筛选已实时生效，则移除按钮；如需显式触发，则按钮有效
5. "重置"按钮：清空所有筛选条件 + URL query

**修复方案**:
- 移除无效的"查询"按钮（如筛选已实时生效）
- 前端分页改造为后端分页

**验收标准**（引用总纲 2.2）:
- [ ] URL 同步
- [ ] 重置清空所有条件

## 分页

### OUT-38: 分页操作

| 属性 | 内容 |
|:---|:---|
| **角色** | admin, warehouse_manager |
| **触发** | 点击页码/上一页/下一页 |
| **当前状态** | ⚠️ 有缺陷 |

**缺陷**: 前端 `filteredData.slice`

**规范定义**（引用总纲 2.3）:
1. 分页参数传给后端：`GET /outbound?page=2&pageSize=20`
2. 后端返回 `{ list, pagination: { total, page, pageSize } }`
3. 切换页码时 URL 同步：`?page=2`

**修复方案**:
- 后端 `GET /outbound` 已支持分页参数，但前端未传递
- 前端修改 `fetchData` 传递 `page`/`pageSize`
- 封装通用分页 hook

**验收标准**（引用总纲 2.3）:
- [ ] 分页参数传给后端
- [ ] 显示"共 X 条"
- [ ] URL 同步
