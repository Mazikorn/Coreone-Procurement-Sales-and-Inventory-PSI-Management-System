## 数据查看

### LOC-01: 查看库位树

| 属性 | 内容 |
|:---|:---|
| **角色** | admin, warehouse_manager |
| **触发** | 进入页面 |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 进入页面 → 调 `GET /locations/tree` 加载库位层级树
2. 左侧树形结构展示所有库位节点
3. 根节点为"全部库位"，子节点按层级展开

**验收标准**:
- [ ] 树形结构正确渲染
- [ ] 节点层级关系与数据库一致
- [ ] 加载失败时显示错误提示和重试按钮

## 库位管理

### LOC-07: 新建库位

| 属性 | 内容 |
|:---|:---|
| **角色** | admin |
| **触发** | 点击"新建"按钮 |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 点击"新建" → 打开 create 弹窗（size=md）
2. 表单字段：库位编码（必填）、库位名称（必填）、父库位（下拉选择）、总容量（number）、备注
3. 点击"保存" → 校验 → 调 `POST /locations` → 成功关闭弹窗 + 刷新树

**验收标准**:
- [ ] 编码和名称为必填，未填时阻止提交
- [ ] 编码唯一性校验
- [ ] 成功后树形结构自动刷新并定位到新节点

---

### LOC-08: 编辑库位

| 属性 | 内容 |
|:---|:---|
| **角色** | admin |
| **触发** | 点击"编辑"按钮 |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 选中库位节点 → 点击"编辑" → 打开 edit 弹窗（size=md）
2. 弹窗预填充当前库位数据
3. 可修改字段：名称、父库位、总容量、备注
4. 点击"保存" → 调 `PUT /locations/:id` → 成功关闭弹窗 + 刷新树

**验收标准**:
- [ ] 弹窗正确预填充数据
- [ ] 父库位下拉排除当前节点及其子节点（防止循环引用）
- [ ] 提交后树形结构刷新

---

### LOC-09: 删除库位

| 属性 | 内容 |
|:---|:---|
| **角色** | admin |
| **触发** | 点击"删除"按钮 |
| **当前状态** | ✅ 可用 |

**交互步骤**:
1. 选中库位节点 → 点击"删除"
2. 调 `GET /locations/:id/check-deletable` 预检查
3. 打开自定义确认弹窗，显示删除影响
4. 点击"确认删除" → 调 `DELETE /locations/:id` → 成功 toast + 刷新树

**验收标准**:
- [ ] 自定义确认弹窗（非原生 confirm）
- [ ] 有子节点或关联库存时显示影响并禁用删除
- [ ] 删除后树形结构刷新

---
