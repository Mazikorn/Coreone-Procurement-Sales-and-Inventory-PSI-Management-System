| **角色** | admin |
| **触发** | 进入页面 |

## 筛选栏

### USR-05~08: 筛选栏操作

| 属性 | 内容 |
|:---|:---|
| **角色** | admin |
| **触发** | 输入/选择筛选条件 |
| **当前状态** | ⚠️ 有缺陷 |

**缺陷汇总**:
- 全部筛选：URL 不同步
- 筛选生效，但分页为前端 `filteredData.slice`

**规范定义**（引用总纲 2.2）:
1. 搜索框实时搜索（debounce 300ms），按用户名/姓名搜索
2. 状态筛选 onChange 实时生效（启用/停用）
3. 角色筛选 onChange 实时生效
4. "重置"按钮：清空所有筛选条件 + URL query
5. 筛选变化时重置到第 1 页

**修复方案**:
- 筛选条件同步到 URL：`?search=xxx&status=active&role=admin`
- 筛选参数传给后端 `GET /users?search=xxx&status=active&role=admin`
- 封装通用筛选 hook，统一管理 URL 同步

**验收标准**（引用总纲 2.2）:
- [ ] URL 同步
- [ ] 筛选后重置分页
- [ ] 重置清空所有条件

## 分页

### USR-25: 分页操作

| 属性 | 内容 |
|:---|:---|
| **角色** | admin |
| **触发** | 点击页码/上一页/下一页 |
| **当前状态** | ⚠️ 有缺陷 |

**缺陷**: 前端 `filteredData.slice`

**规范定义**（引用总纲 2.3）:
1. 分页参数传给后端：`GET /users?page=2&pageSize=20`
2. 后端返回 `{ list, pagination: { total, page, pageSize } }`
3. 切换页码时 URL 同步：`?page=2`
4. 筛选变化时重置到第 1 页

**修复方案**:
- 后端 `GET /users` 已支持分页参数，前端未传递
- 前端修改 `fetchData` 传递 `page`/`pageSize`
- 封装通用分页 hook

**验收标准**（引用总纲 2.3）:
- [ ] 分页参数传给后端
- [ ] 显示"共 X 条"
- [ ] URL 同步
- [ ] 筛选后重置分页
