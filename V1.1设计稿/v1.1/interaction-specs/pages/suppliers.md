## 筛选栏

### SUP-05~07: 筛选栏操作

| 属性 | 内容 |
|:---|:---|
| **角色** | 全部授权角色 |
| **触发** | 输入/选择筛选条件 |
| **当前状态** | ⚠️ 有缺陷 |

**缺陷汇总**:
- 全部筛选：**URL 不同步**
- 前端分页导致筛选后数据量大的场景体验差

**规范定义**（引用总纲 2.2）:
1. 搜索框实时搜索（debounce 300ms），按名称/联系人搜索
2. 状态下拉 onChange 实时生效
3. "重置"按钮：清空所有筛选条件 + URL query

**修复方案**:
- 使用 `useSearchParams` 同步所有筛选条件到 URL
- 封装通用筛选 hook，统一管理筛选状态

**验收标准**（引用总纲 2.2）:
- [ ] URL 同步
- [ ] 重置清空所有条件
- [ ] 刷新页面后筛选状态保留

## 分页

### SUP-21: 分页操作

| 属性 | 内容 |
|:---|:---|
| **角色** | 全部授权角色 |
| **触发** | 点击页码/上一页/下一页 |
| **当前状态** | ⚠️ 有缺陷 |

**缺陷**: 前端 `filteredData.slice`

**规范定义**（引用总纲 2.3）:
1. 分页参数传给后端：`GET /suppliers?page=2&pageSize=20`
2. 后端返回 `{ list, pagination: { total, page, pageSize } }`
3. 切换页码时 URL 同步：`?page=2`
4. 显示"共 X 条"

**修复方案**:
- 后端 `GET /suppliers` 已支持分页参数，前端未传递
- 前端修改 `fetchData` 传递 `page`/`pageSize`
- 封装通用分页 hook（复用其他页面）

**验收标准**（引用总纲 2.3）:
- [ ] 分页参数传给后端
- [ ] 显示"共 X 条"
- [ ] URL 同步
- [ ] 筛选后重置到第 1 页
