## 统计卡片

### ST-01: 查看进行中统计

| 属性 | 内容 |
|:---|:---|
| **角色** | warehouse_manager |
| **触发** | 进入页面 |
| **当前状态** | ❌ 缺失 |

**缺陷**: `const inProgress = 0` 硬编码

**规范定义**:
1. 进入页面 → 调 `GET /stocktaking/stats` 获取统计数据
2. 显示"进行中"统计卡片，值为真实进行中盘点任务数量
3. 点击卡片可筛选进行中状态

**修复方案**:
```typescript
// 移除硬编码
// const inProgress = 0;

// 改为从 API 获取
const [stats, setStats] = useState({ inProgress: 0, completed: 0, pendingDiff: 0, accuracy: 0 });

useEffect(() => {
  stocktakingApi.getStats().then(res => setStats(res.data));
}, []);
```

**验收标准**:
- [ ] 数字来自后端 API
- [ ] 无硬编码 fallback
- [ ] 空数据时显示 `0`
- [ ] 点击卡片筛选生效

## 分页

### ST-21: 分页操作

| 属性 | 内容 |
|:---|:---|
| **角色** | warehouse_manager |
| **触发** | 点击页码 |
| **当前状态** | ⚠️ 有缺陷 |

**缺陷**: 前端 `filteredData.slice`

**规范定义**（引用总纲 2.3）:
1. 分页参数传给后端：`GET /stocktaking?page=2&pageSize=20`
2. 后端返回 `{ list, pagination: { total, page, pageSize } }`
3. 切换页码时 URL 同步：`?page=2`

**修复方案**:
- 后端 `GET /stocktaking` 已支持分页参数，但前端未传递
- 前端修改 `fetchData` 传递 `page`/`pageSize`
- 封装通用分页 hook

**验收标准**（引用总纲 2.3）:
- [ ] 分页参数传给后端
- [ ] 显示"共 X 条"
- [ ] URL 同步
