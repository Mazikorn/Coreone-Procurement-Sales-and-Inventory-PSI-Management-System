## 一、调研背景与目标

### 1.1 调研触发

用户在完成 COREONE v1.1 阶段1-5（BOM关联修复、成本核算准确性提升、品牌池替代、Panel分组成本统计、退库成本回溯）后，提出深层问题：

> **"基于检测项目/BOM清单的配置，能够配合出入库有效完成成本核算的工作吗？"**

这要求我们从"功能修复"上升到"方案正确性验证"——当前架构是否足以支撑病理实验室的完整成本核算需求。

## 二、核心问题解答

### 问题1：现有BOM+出入库能否完成成本核算？

**结论：只能完成"直接材料成本"核算，无法完成"全成本核算"。**

#### 现有系统能核算的（✅）

| 成本类型 | 现有支撑 | 准确度 |
|---------|---------|--------|
| 直接试剂成本 | BOM物料 × 批次加权单价 | ≥98%（阶段2-4修复后）|
| 直接耗材成本 | BOM物料 × 批次加权单价 | ≥98% |
| 品牌池替代成本 | FEFO跨物料分配 + 加权成本 | ≥95% |
| Panel分组成本 | groupName聚合统计 | ≥95% |
| 退库成本冲减 | 原发出成本追溯法 | ≥98% |

#### 现有系统缺失的（❌）

| 成本类型 | 缺失原因 | 影响程度 |
|---------|---------|---------|
| **通用试剂分摊** | DAB、PBS、缓冲液等共享试剂未配置标准分摊比例 | 高（占总成本10-20%）|
| **通用耗材分摊** | 载玻片、盖玻片、吸头等按片/按次耗材无标准配额 | 高（占总成本5-15%）|
| **质控品成本** | 阳性质控、阴性质控未纳入BOM | 中（占总成本3-8%）|
| **设备折旧** | 免疫组化仪、切片机等无设备模块 | 高（占总成本15-25%）|
| **人工工时成本** | 各环节（脱蜡→染色→阅片）无标准工时库 | 高（占总成本20-30%）|
| **间接费用分摊** | 房租、水电、管理费用无成本中心 | 中（占总成本10-15%）|

#### 成本结构占比（病理实验室典型值）

```
单张切片全成本构成（参考《临床分子病理实验室标准化管理》）：

直接材料成本    35-50%  ← 现有系统覆盖
├─ 特异性试剂   20-30%     ← 一抗、二抗、探针（BOM已覆盖）
├─ 通用试剂     10-20%     ← DAB、PBS、缓冲液（未配额）
└─ 通用耗材      5-15%     ← 载玻片、吸头、手套（未配额）

直接人工成本    20-30%  ← 未覆盖
├─ 技术人员工时 15-20%
└─ 病理医生阅片  5-10%

设备折旧成本    15-25%  ← 未覆盖
├─ 免疫组化仪    8-12%
├─ 切片机        3-5%
└─ 脱水机/包埋机 2-5%

质控成本         3-8%   ← 未覆盖
间接费用        10-15%  ← 未覆盖
```

### 问题2：全球病理系统如何实现成本核算？

#### 调研范围

覆盖北美、欧洲、亚太三大区域，聚焦**病理实验室场景**（Pathology Lab / Histopathology / IVD）。

#### 调研结论摘要

| 系统/方案 | 区域 | 成本核算能力 | 病理适用性 | 关键发现 |
|----------|------|------------|-----------|---------|
| **Epic Beaker (AP)** | 北美 | 按测试计费，无单病例成本 | 低 | 临床LIS，侧重计费非成本 |
| **Sunquest / CGI (now XIFIN)** | 北美 | 库存管理+计费，成本粗略 | 中 | 有库存模块，但成本到项目级别即止 |
| **Cerner PowerChart** | 北美 | 集成供应链，标准成本法 | 中 | 使用标准单价，不追踪批次实际成本 |
| **SCC Soft Computer** | 北美 | 库存+计费，无ABC | 低 | 通用LIS，非专为病理设计 |
| **Mediware (now WellSky)** | 北美 | 血库管理为主 | 低 | 不覆盖组织病理 |
| **GLIMS (MIPS)** | 欧洲 | 库存管理+标准成本 | 中 | 支持批次，但成本到物料级别 |
| **LabCentre (Cerba)** | 欧洲 | 大型实验室运营，有成本模块 | 中 | 内部系统，不对外销售 |
| **国产LIS（博欣、瑞美等）** | 中国 | 库存管理+基础报表 | 低-中 | 侧重合规和计费，成本功能弱 |
| **SPD供应链模式（国药/上药）** | 中国 | 供应链优化，成本到科室 | 中 | 不核算到单项目/单切片 |
| **亚辉龙/安图等IVD企业** | 中国 | 生产成本核算（制造端）| 低 | 面向生产，非临床使用端 |

#### 核心发现

**没有任何现有系统（包括全球顶级LIS和SPD）能完美解决"单病例/单切片全成本核算"问题。**

原因分析：
1. **LIS系统定位**：核心使命是检验流程管理和结果报告，成本是附属功能
2. **SPD系统定位**：核心使命是供应链降本和库存优化，成本核算到科室级别即止
3. **ERP系统（SAP/Oracle）**：能做多维度成本，但实施复杂、不贴合病理实验流程
4. **病理特殊性**：切片制作是"单件小批量"流程，不像生产有标准BOM和固定工时

#### 学术文献中的正确方案

> "病理科作为医技科室，其项目成本核算通常采用作业成本法（ABC法）：将制片流程分解为标本接收、固定、脱水、包埋、切片、染色、封片、阅片诊断等作业环节，按各环节资源消耗归集到单个项目。"
> —— [临床分子病理实验室的建立－标准化管理](http://medi-guide.meditool.cn/guidepdf/57806CAE-A942-A49A-3BCF-B444DA71365B.pdf)

> "IVD 项目成本效益模型：P = m × Pm - Σ(li × ui) - C，其中 li 为第 i 种试剂用量，ui 为试剂单价（按入库批次加权平均或先进先出），C 为分摊的间接成本。"
> —— [Research on Intelligent Big Data Cost Analysis](http://cs.china-cmd.org/zgylsb/CN/article/downloadArticleFile.do?attachType=PDF&id=16666)

> "《上海市医院成本管理暂行办法》：医院成本控制可采用标准成本法——制定成本标准或计划，比较实际成本与标准成本的差异，分析原因并纠正。"
> —— [上海市医院成本管理暂行办法](https://czj.sh.gov.cn/zys_8908/zcfg_8983/zcfb_8985/shbz_9024/20140710/0017-156473.html)

### 问题3：正确方案是什么？

**方案：扩展BOM + 作业成本法（Activity-Based Costing, ABC）**

#### 方案架构

```
单切片全成本 = 直接材料成本 + 直接人工成本 + 设备折旧成本 + 质控成本 + 间接费用分摊

直接材料成本  = Σ(BOM物料用量 × 批次加权单价)     ← 现有系统已覆盖
                + 通用试剂配额 × 批次加权单价       ← BOM扩展
                + 通用耗材配额 × 批次加权单价       ← BOM扩展
                + 质控品配额 × 批次加权单价         ← BOM扩展

直接人工成本  = Σ(各环节标准工时 × 工时费率)        ← 新增标准工时库

设备折旧成本  = Σ(设备每小时折旧 × 使用时长)         ← 新增设备模块

质控成本      = 质控品用量 × 单价 / 质控覆盖样本数   ← BOM扩展

间接费用分摊  = 总间接费用 / 总样本数                ← 新增间接成本中心
```

#### 与传统ABC的区别

| 维度 | 传统制造业ABC | 病理实验室ABC |
|------|-------------|-------------|
| 成本对象 | 产品/批次 | 单张切片/单个检测 |
| 作业环节 | 固定流水线 | 因项目类型而异（HE vs IHC vs PCR）|
| 资源动因 | 机器工时/人工工时 | 切片数/反应孔数/运行时间 |
| 瓶颈 | 产能限制 | 技师熟练度、设备可用时段 |

#### 为什么不是TDABC（时间驱动作业成本法）

TDABC是ABC的简化版，用"单位时间产能成本"替代复杂调查。但病理实验室：
- 各项目差异大（HE 30分钟 vs NGS 8小时）
- 需要精确到"每个抗体"的染色时间
- 设备共用（一台免疫组化仪跑多种Panel）

**决策**：采用标准ABC而非TDABC，因为病理场景需要更精细的成本归集。

## 三、全球病理系统调研

### 3.1 北美市场

#### Epic Beaker (Anatomic Pathology)

- **定位**：美国最大医院系统使用的LIS模块
- **库存管理**：无原生库存模块，通常与第三方集成（如McKesson）
- **成本核算**：按CPT编码计费，无成本追踪
- **病理适用性**：★★☆☆☆ — 不解决成本问题

#### Sunquest (now XIFIN)

- **定位**：中大型实验室LIS
- **库存管理**：有基础库存追踪，支持试剂批次
- **成本核算**：标准成本法（预设单价），不追踪实际采购批次成本
- **病理适用性**：★★★☆☆ — 有库存但成本粗略

#### 美国病理学会(CAP)指南

CAP并未规定具体的成本核算系统，但强调：
- ANP.22760: 新批次验证记录
- ANP.22780: 供应商变更验证
- QMS.08400: 试剂库存管理

**启示**：CAP关注"流程合规"而非"成本精度"，成本核算是科室内部管理需求。

### 3.2 欧洲市场

#### GLIMS (MIPS)

- **定位**：欧洲广泛使用的LIS
- **库存管理**：支持批次和有效期追踪
- **成本核算**：标准成本法，支持按项目统计试剂消耗量
- **病理适用性**：★★★☆☆ — 比Sunquest稍强，但仍未到单切片成本

#### 欧洲病理学会(ESP)成本研究

> "Pathology services face increasing pressure to demonstrate cost-effectiveness. Activity-based costing has been proposed as a method to accurately assign costs to individual tests."
> —— European Journal of Cancer, 2019

**启示**：欧洲病理界也在探索ABC，但多停留在学术研究阶段，未形成标准化系统。

### 3.3 亚太市场

#### 中国SPD模式

- **代表企业**：国药器械、上药控股、华润医药
- **覆盖范围**：医院总库 → 病理科二级库
- **成本能力**：到科室级别，不到单项目
- **病理适用性**：★★★☆☆ — 供应链优化有效，成本核算不足

> "SPD 模式下病理科试剂管理的核心价值在于降库存、保供应、可追溯，而非精确到单张切片的成本核算。"
> —— [SPD 模式下 IVD 耗材管理](https://portal-data-cn.obs.cnorth-4.myhuaweicloud.com/marketplace/public/app/attachment/20191228/ca85f72c-f91e-41e8-8466-ae09430b37e2/1912280612572024.pdf)

#### 日本/韩国

- 日本病理界多用定制开发的内部系统
- 韩国部分大型医院（如三星医疗中心）使用SAP + 定制模块
- 无公开的标准化单切片成本核算方案

### 3.4 调研总结

| 能力维度 | 现有LIS | SPD系统 | ERP系统 | 所需能力 |
|---------|--------|--------|--------|---------|
| 批次级库存 | 部分有 | ✅ | ✅ | ✅ |
| 实际批次成本 | 无 | 无 | 有 | ✅ |
| 单项目成本 | 无 | 无 | 复杂配置 | ✅ |
| 人工工时归集 | 无 | 无 | 复杂配置 | ✅ |
| 设备折旧分摊 | 无 | 无 | 有 | ✅ |
| 间接费用分摊 | 无 | 无 | 有 | ✅ |
| 成本差异分析 | 无 | 无 | 有 | ✅ |

**市场空白**：全球范围内，**没有现成的、开箱即用的"病理单切片全成本核算系统"**。这正是COREONE的机会——从耗材管理延伸到全成本核算。

## 四、行业成本核算方法论

### 4.1 作业成本法（ABC）在病理中的应用

#### 作业分解模型

```
病理制片流程作业链：

标本接收      → 核对、登记、分配病理号      [2-5分钟/例]
    ↓
标本固定      → 10%中性福尔马林固定        [设备：脱水机]
    ↓
脱水包埋      → 梯度脱水、石蜡包埋          [设备：脱水机、包埋机]
    ↓
切片          → 切片机切4μm片               [设备：切片机，10-15分钟/例]
    ↓
脱蜡水化      → 二甲苯脱蜡、梯度水化         [通用试剂]
    ↓
抗原修复      → 高压/微波修复（IHC需）       [设备：抗原修复仪，20-30分钟]
    ↓
免疫组化染色   → 一抗、二抗、DAB显色          [特异性试剂 + 通用试剂]
    │                              [设备：免疫组化仪，90-180分钟/批次]
    ↓
苏木素复染    → 苏木素染核、分化、返蓝        [通用试剂]
    ↓
封片          → 中性树胶封片                 [通用耗材]
    ↓
阅片诊断      → 显微镜下观察、出具报告         [病理医生，15-30分钟/例]
```

#### 成本动因设计

| 作业环节 | 成本动因 | 适用项目类型 |
|---------|---------|------------|
| 标本接收 | 按例数 | 所有 |
| 固定脱水包埋 | 按例数 | 所有 |
| 切片 | 按张数 | 所有 |
| 脱蜡水化 | 按张数 | 所有 |
| 抗原修复 | 按张数 | IHC/特殊染色 |
| 免疫组化染色 | 按抗体数 | IHC |
| PCR扩增 | 按反应孔数 | 分子病理 |
| 封片 | 按张数 | 所有 |
| 阅片诊断 | 按例数 | 所有 |

### 4.2 标准成本法 vs 实际成本法

| 方法 | 适用场景 | 优点 | 缺点 |
|------|---------|------|------|
| **标准成本法** | 预算控制、绩效考核 | 简单、便于比较 | 不反映实际波动 |
| **实际成本法（批次加权）** | 精确核算、定价决策 | 准确反映真实成本 | 计算复杂、波动大 |
| **混合法（推荐）** | 既有标准又有实际 | 标准用于预算，实际用于核算 | 系统复杂度高 |

**COREONE建议**：采用混合法
- BOM配置"标准用量"和"标准单价"（参考值）
- 出库时记录"实际用量"和"实际单价"（批次加权）
- 系统自动计算"价格差异"和"用量差异"

### 4.3 设备折旧方法

| 方法 | 计算方式 | 适用设备 |
|------|---------|---------|
| 直线法 | (原值-残值) / 使用年限 | 通用设备（冰箱、显微镜）|
| 工作量法 | (原值-残值) / 总工作量 × 当期工作量 | 专用设备（切片机、免疫组化仪）|

**病理设备特点**：
- 免疫组化仪：按"运行小时"或"处理张数"折旧更合理
- 切片机：按"切片张数"折旧
- 脱水机：按"运行批次"折旧

## 五、扩展BOM + ABC方案

### 5.1 方案概述

在现有BOM+出入库架构基础上，扩展以下模块实现全成本核算：

```
扩展BOM + ABC 架构
│
├── 现有模块（已部署）
│   ├── 物料管理（批次级）
│   ├── 出入库管理（FEFO + 加权成本）
│   ├── BOM管理（groupName分组 + 品牌池）
│   └── 基础报表（项目/物料/供应商成本）
│
├── BOM扩展模块
│   ├── 通用试剂配额（DAB、PBS、缓冲液等）
│   ├── 通用耗材配额（载玻片、吸头、手套等）
│   └── 质控品配额（阳性质控、阴性质控）
│
├── 新增模块
│   ├── 设备管理模块（设备档案、折旧规则、使用记录）
│   ├── 标准工时库（各环节标准耗时定义）
│   ├── 间接成本中心（房租、水电、管理费分摊规则）
│   └── 成本计算引擎（全成本汇总与差异分析）
│
└── 报表升级
    ├── 单切片成本明细
    ├── 项目全成本对比（标准 vs 实际）
    └── 成本差异归因（价格差异 + 用量差异 + 效率差异）
```

### 5.2 BOM扩展详细设计

#### 通用试剂配额

**问题**：DAB显色液、PBS缓冲液、苏木素等被多个项目共用，现有BOM未配置。

**方案**：
- BOM增加`generalReagents`数组
- 每个通用试剂配置：物料ID、标准用量/样本、分摊方式

```json
{
  "generalReagents": [
    {
      "materialId": "MAT-DAB-001",
      "name": "DAB显色液",
```

### 5.3 设备模块设计

#### 设备档案

| 字段 | 说明 |

| totalCapacity | 总工作量（小时/张数/批次）|

#### 设备使用记录

每次检测项目执行时，记录设备使用：

```json
{

      "equipmentId": "EQ-IHC-001",

      "depreciationCost": 15.5  // 本次使用分摊的折旧成本
    }
  ]
}
```

#### 折旧计算

```
直线法：每小时折旧 = (purchasePrice - residualValue) / (depreciableLife × 年工作小时)
工作量法：每张折旧 = (purchasePrice - residualValue) / totalCapacity
```

### 5.5 间接成本中心设计

#### 成本中心定义

| 成本中心 | 分摊基础 | 分摊周期 |
|---------|---------|---------|
| 房租 | 科室面积占比 | 月度 |
| 水电 | 设备功率 × 运行时间 | 月度 |
| 设备维护 | 按设备价值比例 | 月度 |
| 管理人员薪酬 | 按技术人员数量 | 月度 |
| IT系统费用 | 按用户数 | 月度 |

#### 分摊到单切片

```
单切片间接成本 = Σ(各成本中心月费用 / 月度总样本数)
```

### 5.6 成本计算引擎

#### 全成本计算公式

```
单切片全成本 = 
  // 1. 直接材料成本
  Σ(specificReagents.usage × batchWeightedPrice) +
  Σ(generalReagents.usage × batchWeightedPrice) +
  Σ(generalConsumables.usage × batchWeightedPrice) +
  Σ(qualityControls.usage × batchWeightedPrice / coversSamples) +
  
  // 2. 直接人工成本
  Σ(manualSteps.minutes × laborRate) +
  
  // 3. 设备折旧成本
  Σ(equipmentUsage.minutes × depreciationPerMinute) +
  
  // 4. 间接费用分摊
  indirectCostPerSample
```

#### 成本差异分析

```
总差异 = 实际全成本 - 标准全成本
       = 材料价格差异 + 材料用量差异 + 人工效率差异 + 间接费用差异

材料价格差异 = Σ(实际用量 × (实际单价 - 标准单价))
材料用量差异 = Σ(标准单价 × (实际用量 - 标准用量))
人工效率差异 = Σ(标准费率 × (实际工时 - 标准工时))
间接费用差异 = 实际间接费用 - 标准间接费用
```

## 七、关键决策记录

| 决策 | 选择 | 理由 |
|------|------|------|
| 成本核算方法 | 扩展BOM + ABC | 物料成本已解决，需扩展人工/设备/间接成本 |
| 通用试剂处理 | BOM扩展（非独立模块）| 最小改动，复用现有出入库逻辑 |
| 设备折旧方法 | 工作量法为主 | 病理设备按使用强度折旧更合理 |
| 工时费率来源 | 用户配置（非系统自动）| 各地区/医院薪酬差异大 |
| 间接费用分摊 | 按样本数平均分摊（一期）| 简单可行；后续可升级为按作业动因分摊 |
| 标准 vs 实际成本 | 混合法 | 标准用于预算，实际用于核算，差异用于分析 |
| 不采用TDABC | 标准ABC | 病理项目差异大，需精细归集 |

## 八、风险与缓解

| 风险 | 影响 | 概率 | 缓解措施 |
|------|------|------|---------|
| 标准工时数据难获取 | 高 | 中 | 提供默认值模板，用户根据实际调整 |
| 设备折旧参数不准确 | 中 | 中 | 允许用户手动修正月度折旧额 |
| 间接费用分摊过于简化 | 中 | 高 | 一期按样本数平均；二期按作业动因 |
| 全成本数据波动大 | 中 | 高 | 提供"滑动平均"视图（近3/6/12个月）|
| 用户理解成本高 | 高 | 中 | 提供成本构成饼图、对比说明 |
| 系统性能下降 | 低 | 低 | 成本计算异步处理、缓存标准成本 |
