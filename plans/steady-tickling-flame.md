# Phase 6.6 全方位测试计划（扩展BOM + ABC全成本核算）

## 背景

Phase 6（扩展BOM + ABC全成本核算）已完成 6.1-6.5 的全部开发，包含 4 个全新功能模块（BOM扩展配额、设备管理、标准工时库、间接成本中心）和 1 个核心计算引擎（全成本计算）。这些模块相互依赖、计算逻辑复杂，需要系统化测试确保数据准确性和功能稳定性。

---

## 一、测试目标

| # | 目标 | 验证方式 |
|---|------|---------|
| 1 | **产品目标达成** | 单切片全成本 = 材料 + 人工 + 设备 + 质控 + 间接，数据准确可支撑经营决策 |
| 2 | **功能稳定性** | 所有 CRUD 操作正常，边界条件处理正确，并发操作安全 |
| 3 | **计算准确性** | 手工计算 vs 系统计算误差 < 1%，多批次/替代物料/跨月份场景正确 |
| 4 | **集成正确性** | BOM扩展配额 → 出库 → 成本计算 → 报表展示，端到端链路通畅 |
| 5 | **交互体验** | 6 个 Tab 切换正常，表单校验友好，加载状态清晰，错误恢复可用 |

## 七、风险与应对

| 风险 | 影响 | 应对 |
|------|------|------|
| `node:sqlite` 在测试环境不稳定 | 集成测试无法运行 | 使用 `:memory:` 数据库 + `--experimental-sqlite` 标志 |
| 成本计算涉及多表JOIN，性能慢 | 报表加载 > 5秒 | 添加索引（project_cost_details.outbound_id, cost_month） |
| 设备折旧计算有浮点精度问题 | 误差 > 1% | 统一使用 DECIMAL(18,4)，计算时四舍五入到分 |
| 间接成本分摊基数为0 | 除零错误 | 后端校验：allocation_base_value > 0 |
| 多用户并发修改BOM | 数据不一致 | 使用数据库事务 + 乐观锁（version字段） |
