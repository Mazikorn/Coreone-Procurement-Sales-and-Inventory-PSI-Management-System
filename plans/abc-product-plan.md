# COREONE ABC 成本核算 — 产品规划

> **核心价值**: 成本透明化 — 让每张切片的成本清晰可见
> **设计原则**: 用户故事驱动，非技术架构驱动
> **参考文档**:
> - [三方案归档](../research/abc-three-plans-archive.md)
> - [市场调研报告](../research/abc-market-research.md)
> - [方案二系统规划](abc-deep-integration-plan.md)

## 一、用户故事与场景

### 1.1 病理科主任/管理者

**核心问题**：我们科每个月花多少钱？赚多少钱？哪些项目亏钱？

| 用户故事 | 验收标准 |
|---------|---------|
| 作为主任，我想看本月成本概况，以便知道这个月花了多少钱 | 一屏展示总成本、总收入、总利润、利润率 |
| 作为主任，我想看哪些项目赚钱哪些亏钱，以便调整检测策略 | 项目盈利性排名，利润率颜色标签 |
| 作为主任，我想看成本趋势，以便发现异常波动 | 月度成本趋势折线图 |
| 作为主任，我想导出成本报表，以便在科室会议上汇报 | 支持 Excel/PDF 导出 |

**使用频率**：每周 1-2 次，月度会议前集中查看

### 1.2 仓库管理员

**核心问题**：出库的时候，我知道这次出库花了多少钱吗？

| 用户故事 | 验收标准 |
|---------|---------|
| 作为仓管，我想在出库时看到这次出库的总成本，以便心中有数 | 出库弹窗显示成本预览面板 |
| 作为仓管，我想看到每张切片的成本，以便理解成本构成 | 成本预览显示单张切片成本 |
| 作为仓管，我想看到收费金额和利润，以便了解这次出库的价值 | 成本预览显示收费和利润 |

**使用频率**：每天多次，每次出库时

### 1.3 财务人员

**核心问题**：这个月的成本核算准确吗？收费有没有漏？

| 用户故事 | 验收标准 |
|---------|---------|
| 作为财务，我想按月查看成本明细，以便核对账目 | 月度成本明细表，支持按项目/物料筛选 |
| 作为财务，我想对比成本和收费，以便发现漏收错收 | 收费 vs 成本对比表 |
| 作为财务，我想查看成本构成，以便分析成本结构 | 成本结构饼图（材料/人工/设备/间接） |
| 作为财务，我想导出成本数据，以便做进一步分析 | 支持 Excel 导出 |

**使用频率**：每月 1-2 次，月末核算时

### 1.4 技术人员

**核心问题**：BOM 配置得对吗？成本计算准确吗？

| 用户故事 | 验收标准 |
|---------|---------|
| 作为技术，我想为 BOM 配置收费标准，以便系统能自动计算收费 | BOM 详情页增加收费标准 Tab |
| 作为技术，我想查看 BOM 的标准成本，以便验证配置是否正确 | BOM 列表显示标准成本和利润率 |
| 作为技术，我想配置作业中心和成本动因，以便 ABC 计算更精确 | 作业中心/成本动因管理页面 |

**使用频率**：配置阶段集中使用，日常偶尔调整

## 三、数据流设计（用户视角）

### 3.1 配置阶段（技术人员）

```
技术人员操作：
1. 配置作业中心（已有，8个）
2. 配置成本动因（已有，7种）
3. 为每个 BOM 配置作业关联（BOM详情 → 作业关联Tab）
4. 为每个 BOM 配置收费标准（BOM详情 → 收费标准Tab）
5. 按月录入成本池数据（成本池管理页面）

系统自动：
- 计算每个作业中心的动因费率
- 计算每个 BOM 的标准切片成本
- 计算每个 BOM 的标准利润率
```

### 3.2 运行阶段（仓库管理员）

```
仓库管理员操作：
1. 出库登记 → 选择项目 → 选择 BOM → 输入样本数
2. 系统实时显示成本预览（材料+作业+收费+利润）
3. 确认出库

系统自动：
- 计算材料成本（已有，批次加权价）
- 计算作业成本（新增，ABC引擎）
- 匹配收费标准（新增，阶梯定价）
- 写入 outbound_abc_details（新增）
- 更新 outbound_records（新增字段）
```

### 3.3 分析阶段（主任/财务）

```
主任/财务操作：
1. 打开成本看板 → 看到本月概况
2. 点击某个项目 → 看到切片成本明细
3. 切换到盈利分析 → 看到项目/病例/BOM三维度
4. 导出报表

系统自动：
- 从 outbound_abc_details 聚合数据
- 计算各维度的汇总指标
- 生成趋势图表
```

## 五、实施计划

### Phase 1: 收费标准 + BOM 映射（Week 1）

**目标**: 让 BOM 能关联收费标准，为后续计算做准备

**交付物**:
1. DatabaseManager.ts: boms 表增加 fee_standard_id, fee_category 字段
2. bom-v1.1.ts: BOM CRUD 支持收费标准字段
3. BOM 详情页增加收费标准 Tab
4. BOM 列表增加收费和利润率列
5. 种子数据：为预置 BOM 配置收费标准映射

**验收标准**:
- BOM 详情页可以配置收费标准
- 收费规则预览正确显示
- 标准切片成本和利润率自动计算

### Phase 2: ABC 计算引擎 + 出库集成（Week 2-3）

**目标**: 出库时自动计算 ABC 成本和收费

**交付物**:
1. DatabaseManager.ts: 新增 outbound_abc_details 表
2. cost-calculator.ts: 扩展 calculateSlideCost，新增 calculateFeeAmount
3. outbound-v1.1.ts: POST /bom 嵌入 ABC 计算 + 收费匹配
4. abc-v1.1.ts: 新增 slide-cost, fee-mapping 端点
5. 出库弹窗增加成本预览面板
6. 出库列表增加 ABC 成本、收费、利润列
7. 单元测试

**验收标准**:
- BOM 出库自动计算并保存 ABC 成本
- 出库时可预览成本、收费、利润
- 单张切片成本误差 < 0.01 元
- 阶梯定价和封顶机制正确
- 单元测试覆盖率 > 85%

### Phase 3: 成本看板 + 盈利分析（Week 4-5）

**目标**: 管理层能看到成本全貌和盈利性

**交付物**:
1. 成本看板页面（/abc/dashboard）
2. 切片成本明细页面（/abc/slide-cost）
3. 盈利分析页改造（三维度视图）
4. 成本瀑布图组件
5. 侧边栏增加"成本分析"分组

**验收标准**:
- 成本看板一屏展示总成本、总收入、总利润、利润率
- 切片成本明细表正确展示各 BOM 的成本构成
- 盈利分析支持项目/病例/BOM 三维度切换
- 成本瀑布图正确展示作业成本构成

### Phase 4: 趋势分析 + 报表优化（Week 6）

**目标**: 支持历史回溯和报表导出

**交付物**:
