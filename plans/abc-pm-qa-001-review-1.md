## 第一层：意图锁定（生成前）

### VibeContract

#### 业务意图

作为**病理科主任/管理者**，我希望**看到每张切片的完整成本构成和盈利性**，以便**做出正确的资源配置和定价决策**。

作为**仓库管理员**，我希望**出库时看到这次出库的成本和收费**，以便**心中有数**。

作为**财务人员**，我希望**按月核对成本和收费**，以便**发现漏收错收**。

作为**技术人员**，我希望**为 BOM 配置收费标准和作业关联**，以便**系统自动计算成本**。

#### 数据契约

| 字段 | 类型 | 来源 | 约束 | 示例 |
|------|------|------|------|------|
| material_cost | number | 出库时计算 | >= 0 | 125.50 |
| activity_cost | number | ABC 计算引擎 | >= 0 | 62.50 |
| total_cost | number | material + activity | >= 0 | 188.00 |
| cost_per_slide | number | total / slide_count | >= 0 | 18.80 |
| fee_amount | number | 收费标准匹配 | >= 0 | 525.00 |
| profit | number | fee - cost | 可为负 | 337.00 |
| profit_rate | number | profit / fee | -1 ~ 1 | 0.642 |

#### 边界契约

1. **空值处理**：无 BOM 出库时，activity_cost = 0，仅记录 material_cost
2. **降级策略**：成本池为空时，按 三级降级（当月 → 上月 → BOM标准）
3. **事务隔离**：ABC 计算失败不阻断出库事务
4. **幂等性**：出库编辑时，先删旧 ABC 记录再重新计算
5. **权限控制**：读操作 admin/finance/pathologist，写操作 admin/finance

#### 异常契约

| 场景 | 处理方式 |
|------|---------|
| ABC 计算失败 | try-catch 包裹，记录日志，出库继续 |
| 收费标准未配置 | fee_amount = 0，前端显示"未配置"警告 |
| 成本池为空 | 降级使用上月数据或 BOM 标准成本 |
| 出库回滚 | ABC 记录同步删除 |
| BOM 无作业关联 | activity_cost = 0，仅记录 material_cost |

#### 验收标准

- [ ] 出库时自动计算 ABC 成本，响应时间增加 < 100ms
- [ ] 单张切片成本误差 < 0.01 元
- [ ] 阶梯定价和封顶机制正确
- [ ] 成本看板一屏展示总成本、总收入、总利润、利润率
- [ ] 切片成本明细支持展开查看作业成本瀑布图
- [ ] 盈利分析支持项目/病例/BOM 三维度
- [ ] 收费对照按出库记录逐条展示
- [ ] 权限控制正确

### 对抗性提示

#### 边界 1：阶梯定价跨越边界

**场景**：IHC 检测收费规则是"前3项205元，第4-12项210元"，如果一次出库 5 项，应该收 3×205 + 2×210 = 1035 元

**风险**：如果阶梯计算逻辑错误，可能收 5×205 = 1025 元（少收 10 元）

**测试用例**：
```
输入：category='ihc', quantity=5
预期：3×205 + 2×210 = 1035
```

#### 边界 2：封顶机制

**场景**：FISH 收费封顶 3600 元，如果一次出库 4 个探针，应该收 3600 元（不是 4×1200=4800）

**风险**：如果封顶逻辑错误，可能多收 1200 元

**测试用例**：
```
输入：category='fish', quantity=4
预期：3600（封顶）
```

#### 边界 3：成本池降级策略

**场景**：当月成本池为空，上月也为空，需要降级使用 BOM 标准成本

**风险**：如果降级逻辑错误，activity_cost = 0，导致成本严重低估

**测试用例**：
```
输入：activityCenterId='IHC', month='2026-06'（当月无数据）
预期：使用 BOM 标准成本中的默认费率
```
